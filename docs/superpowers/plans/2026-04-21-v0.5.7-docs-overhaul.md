```bash
#!/usr/bin/env bash
# Render Mermaid (.mmd) sources in docs/assets/ to matching .svg files.
#
# Uses mermaid-cli via npx — no global install required.
# Run this manually after editing any .mmd file.
#
# Usage: ./scripts/render-diagrams.sh

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_ROOT="$(dirname "$SCRIPT_DIR")"
ASSETS_DIR="$PROJECT_ROOT/docs/assets"

if [ ! -d "$ASSETS_DIR" ]; then
    echo "error: $ASSETS_DIR does not exist" >&2
    exit 1
fi

shopt -s nullglob
mmd_files=("$ASSETS_DIR"/*.mmd)

if [ ${#mmd_files[@]} -eq 0 ]; then
    echo "No .mmd files in $ASSETS_DIR — nothing to render."
    exit 0
fi

for src in "${mmd_files[@]}"; do
    out="${src%.mmd}.svg"
    echo "→ rendering $(basename "$src") → $(basename "$out")"
    npx --yes @mermaid-js/mermaid-cli \
        --input "$src" \
        --output "$out" \
        --backgroundColor transparent
done

echo "Done."
```

## Core concepts in 60 seconds

- **`Theme`** — the raw, sparse, TOML-shaped definition a preset or file loads into memory. Fields are `Option<T>` because presets can omit almost anything.
- **`ResolvedTheme`** — the fully-populated variant `Theme::resolve(mode)` produces, with every field guaranteed to have a value. This is what your UI code actually reads from.
- **Preset** — a named, bundled theme (e.g. `catppuccin-mocha`, `kde-breeze`, `macos-sonoma`). Load with `Theme::preset("name")`.
- **Connector** — a small crate that maps `ResolvedTheme` onto a specific GUI framework's theming system. You depend on one of these, not on `native-theme` directly.

Cross-platform native theme detection and loading for Rust GUI applications.

<img alt="native-theme crate relations" src="https://raw.githubusercontent.com/tiborgats/native-theme/main/docs/assets/crate-relations.svg" width="640"/>

*Most apps use a connector crate (`native-theme-gpui` or `native-theme-iced`).

## Quick start

```rust
use native_theme::prelude::*;

let sys = SystemTheme::from_system()?;
let active = sys.pick(sys.mode);
// active is a &ResolvedTheme — access active.defaults.background_color, etc.
```

## Core concepts

- **`Theme`** — sparse TOML-shaped definition (fields are `Option<T>`)
- **`ResolvedTheme`** — fully-populated variant; every field has a value
- **`ColorMode`** — `Light` or `Dark`
- **`resolve(mode)`** — applies inheritance + validation; produces `ResolvedTheme`

## Common recipes

### Load a bundled preset
```rust
let theme = Theme::preset("catppuccin-mocha")?;
let resolved = theme.resolve(ColorMode::Dark)?;
```

### Load the OS theme (async for portal-based DEs)
```rust
let sys = native_theme::SystemTheme::from_system_async().await?;
```

### Custom TOML overlay
```rust
let sys = native_theme::SystemTheme::from_system()?;
let customized = sys.with_overlay(include_str!("tweaks.toml"))?;
```
