<objective>
Implement Linux freedesktop icon theme lookup that resolves IconRole variants to SVG bytes from the user's active desktop icon theme (Adwaita, Breeze, Papirus, etc.) using the freedesktop-icons crate, with a full fallback chain: active theme -> hicolor -> bundled Material SVGs.

Purpose: Linux users get platform-native icons from their installed icon themes instead of only bundled fallback icons.
Output: freedesktop.rs module with load_freedesktop_icon() function, system-icons Cargo feature, tests.
</objective>

<task type="auto">
  <name>Task 1: Add freedesktop-icons dependency and create freedesktop.rs module</name>
  <files>native-theme/Cargo.toml, native-theme/src/freedesktop.rs</files>
  <action>
1. In native-theme/Cargo.toml, add the system-icons feature and the freedesktop-icons dependency:

   Under [features], add:
   ```toml
   system-icons = ["dep:freedesktop-icons", "material-icons"]
   ```
   Note: system-icons implies material-icons so the bundled fallback chain always works (per research recommendation and success criteria).

   Add a new platform-gated dependency section:
   ```toml
   [target.'cfg(target_os = "linux")'.dependencies]
   freedesktop-icons = { version = "0.4", optional = true }
   ```

2. Create native-theme/src/freedesktop.rs with the following structure:

   - `detect_theme() -> String` -- calls `freedesktop_icons::default_theme_gtk()`, falls back to "hicolor"
   - `find_icon(name: &str, theme: &str, size: u16) -> Option<PathBuf>` -- two-pass lookup: first tries the plain name via `freedesktop_icons::lookup(name).with_theme(theme).with_size(size).force_svg().find()`, then if None tries `"{name}-symbolic"` (needed for Adwaita which stores action icons only as symbolic variants)
   - `pub fn load_freedesktop_icon(role: IconRole) -> Option<IconData>` -- the public entry point:
     a. Call `icon_name(IconSet::Freedesktop, role)` to get the freedesktop name
     b. If Some(name): call `find_icon(name, &detect_theme(), 24)`, read the file with `std::fs::read()`, return `IconData::Svg(bytes)`
     c. If the icon_name returned None OR find_icon returned None OR fs::read failed: fall back to `bundled_icon_svg(IconSet::Material, role).map(|b| IconData::Svg(b.to_vec()))`

   The entire module must be gated with `#[cfg(target_os = "linux")]` at the top-level (the module declaration in lib.rs will also have cfg gates).

   Use `crate::` imports for `IconData`, `IconRole`, `IconSet`, `icon_name`, `bundled_icon_svg`.

   Add inline tests within `#[cfg(test)] mod tests`:
   - `load_icon_returns_some_for_dialog_error`: calls `load_freedesktop_icon(IconRole::DialogError)`, asserts `is_some()`, checks the result contains `<svg` in the bytes (will work via either system theme or bundled fallback since system-icons implies material-icons)
   - `load_icon_notification_uses_bundled_fallback`: calls `load_freedesktop_icon(IconRole::Notification)`, asserts `is_some()` (Notification has no freedesktop name, so goes straight to bundled Material which has it). Verify it is SVG data.
   - `load_icon_returns_svg_variant`: calls `load_freedesktop_icon(IconRole::ActionCopy)`, pattern matches result to `IconData::Svg(_)`, asserts match succeeds
   - `detect_theme_returns_non_empty`: calls `detect_theme()`, asserts string is not empty (always at least "hicolor")
   - `find_icon_nonexistent_returns_none`: calls `find_icon("totally-nonexistent-icon-xyz", "hicolor", 24)`, asserts `is_none()`

   All tests must be gated with `#[cfg(target_os = "linux")]` (they are already inside the linux-gated module).
   All tests must also have `#[cfg(feature = "system-icons")]` since they use freedesktop_icons.

   Do NOT use `.with_cache()` on the lookup builder (library crate should not pollute global cache).
  </action>
