<task type="auto">
  <name>Task 1: TOML round-trip integration tests</name>
  <files>tests/serde_roundtrip.rs</files>
  <action>
Create tests/serde_roundtrip.rs as an integration test file (in the tests/ directory, NOT src/).

Use `use native_theme::*;` to import all public types. Use `toml` crate directly for serialization.

Implement the following tests:

**test: round_trip_full_theme**
- Construct a NativeTheme with name "Test Theme"
- Populate BOTH light and dark ThemeVariants
- Set at least one field in every sub-struct: core.accent, core.background, core.foreground, primary.background, secondary.background, status.danger, status.success, interactive.selection, interactive.link, panel.sidebar, panel.tooltip, component.button, component.input, fonts.family, fonts.size, fonts.mono_family, fonts.mono_size, geometry.radius, geometry.frame_width, spacing.m, spacing.l
- Serialize to TOML string with toml::to_string_pretty
- Deserialize back with toml::from_str
- Assert name equality
- Assert every set field in light variant matches the deserialized light variant
- Assert every set field in dark variant matches the deserialized dark variant
- This validates SERDE-01 (TOML mapping 1:1 to field names) and TEST-01 (round-trip for all types)

**test: round_trip_preserves_all_36_color_fields**
- Construct a ThemeVariant where ALL 36 color fields are set to distinct Rgba values
- Wrap in NativeTheme, serialize to TOML, deserialize back
- Assert every single one of the 36 fields matches
- This is the exhaustive round-trip test

**test: sparse_toml_deserializes**
- Define a minimal TOML string with only name and light.colors.core.accent set
- Deserialize into NativeTheme
- Assert name is correct
- Assert light.colors.core.accent has the expected value
- Assert all other fields are None/default (e.g., light.colors.core.background is None, dark is None)
- This validates success criterion #3 (sparse TOML)

**test: very_sparse_toml_name_only**
- TOML string: `name = "Empty"`
- Deserialize succeeds
- light and dark are both None

**test: serialization_skips_none_fields**
- Construct NativeTheme with only light.colors.core.accent set
- Serialize to TOML string
- Assert the TOML string contains "accent"
- Assert the TOML string does NOT contain "background" (None fields skipped)
- Assert the TOML string does NOT contain "[light.colors.status]" (empty sub-structs skipped)
- Assert the TOML string does NOT contain "[light.fonts]" (empty fonts section skipped)
- This validates SERDE-02 (skip_serializing_if)

**test: toml_structure_is_human_readable**
- Serialize a theme with a few fields set
- Assert the TOML contains expected section headers: `[light.colors.core]`
- Assert fields appear under correct sections
- Print the TOML to stdout (useful for manual inspection during development)

**test: rgba_hex_in_toml**
- TOML string with `accent = "#3daee9"` and `shadow = "#00000040"`
- Deserialize and verify Rgba values (accent has a=255, shadow has a=0x40=64)
- Serialize back and verify hex strings are lowercase

<task type="auto">
  <name>Task 2: Merge behavior integration tests and trait compile-time assertions</name>
  <files>tests/merge_behavior.rs</files>
  <action>
Create tests/merge_behavior.rs as an integration test file.

Use `use native_theme::*;` to import all public types.

Implement the following tests:

**test: merge_overlay_replaces_none_with_some**
- Create a base ThemeVariant with core.background set
- Create an overlay ThemeVariant with core.accent set (but NOT core.background)
- Call base.merge(&overlay)
- Assert base.core.accent is now the overlay value (was None, replaced by Some)
- Assert base.core.background is unchanged (overlay was None)

**test: merge_overlay_replaces_some_with_some**
- Create base with core.accent = Some(red)
- Create overlay with core.accent = Some(blue)
- After merge, base.core.accent == Some(blue) (overlay wins)

**test: merge_preserves_base_when_overlay_empty**
- Create a populated base ThemeVariant
- Create an empty overlay (default)
- After merge, base is unchanged

**test: merge_native_theme_light_dark**
- Create base NativeTheme with light variant
- Create overlay NativeTheme with dark variant
- After merge, base has both light (from base) and dark (from overlay)
- Name stays as base name (not replaced)

**test: merge_native_theme_deep_merge_variants**
- Create base NativeTheme with light.colors.core.background set
- Create overlay NativeTheme with light.colors.core.accent set
- After merge, base.light has BOTH background (from base) and accent (from overlay)

**test: merge_fonts_geometry_spacing**
- Test merge on ThemeFonts: overlay family replaces, base size preserved
- Test merge on ThemeGeometry: overlay radius replaces, base frame_width preserved
- Test merge on ThemeSpacing: overlay m replaces, base l preserved

**test: merge_chained_multiple_overlays**
- Create a base, overlay1, overlay2
- base.merge(&overlay1); base.merge(&overlay2);
- Verify final state reflects last-wins for fields set in both overlays

**test: is_empty_all_structs**
- ThemeColors::default().is_empty() == true
- CoreColors::default().is_empty() == true
- ThemeFonts::default().is_empty() == true
- ThemeGeometry::default().is_empty() == true
- ThemeSpacing::default().is_empty() == true
- ThemeVariant::default().is_empty() == true

**test: trait_assertions_send_sync**
- Compile-time assertion function: `fn assert_send_sync<T: Send + Sync>() {}`
- Call for: Rgba, NativeTheme, ThemeVariant, ThemeColors, CoreColors, ActionColors, StatusColors, InteractiveColors, PanelColors, ComponentColors, ThemeFonts, ThemeGeometry, ThemeSpacing, Error

**test: trait_assertions_default_clone_debug**
- Compile-time assertion function: `fn assert_default_clone_debug<T: Default + Clone + std::fmt::Debug>() {}`
- Call for all public structs listed above (except Error which is not Default/Clone)
- Separately verify Error is Debug

**test: realistic_theme_layering_scenario**
- Simulate a real use case: "base preset" with many fields set, "user override" with just accent and font family
- Merge user override onto base
- Verify accent and font changed, everything else from base preserved
- Serialize the merged result to TOML and verify it looks correct
