The macro must handle the case where either `option` or `nested` is omitted (e.g., CoreColors has only option fields, ThemeColors has only nested fields). See the research document's Code Examples section for the exact macro_rules! pattern.

  <files>src/color.rs</files>
  <behavior>
    - Rgba::rgb(61, 174, 233) creates Rgba { r: 61, g: 174, b: 233, a: 255 }
    - Rgba::rgba(61, 174, 233, 128) creates Rgba { r: 61, g: 174, b: 233, a: 128 }
    - "#3daee9".parse::<Rgba>() == Ok(Rgba::rgb(61, 174, 233))
    - "#3daee980".parse::<Rgba>() == Ok(Rgba::rgba(61, 174, 233, 128))
    - "3daee9".parse::<Rgba>() == Ok(Rgba::rgb(61, 174, 233)) (no # prefix)
    - "#abc".parse::<Rgba>() == Ok(Rgba::rgb(0xaa, 0xbb, 0xcc)) (3-char shorthand, each digit doubled)
    - "#abcd".parse::<Rgba>() == Ok(Rgba::rgba(0xaa, 0xbb, 0xcc, 0xdd)) (4-char shorthand)
    - "#AABBCC".parse::<Rgba>() == Ok(Rgba::rgb(0xaa, 0xbb, 0xcc)) (uppercase)
    - "".parse::<Rgba>() is Err (empty string)
    - "#gggggg".parse::<Rgba>() is Err (invalid hex chars)
    - "#12345".parse::<Rgba>() is Err (5 chars -- invalid length)
    - Rgba::rgb(61, 174, 233).to_string() == "#3daee9" (lowercase, no alpha when 255)
    - Rgba::rgba(61, 174, 233, 128).to_string() == "#3daee980" (includes alpha when not 255)
    - Rgba round-trips through serde_json/toml as a hex string
    - Rgba::rgb(0, 0, 0).to_f32_array() == [0.0, 0.0, 0.0, 1.0]
    - Rgba::rgba(255, 255, 255, 0).to_f32_array() == [1.0, 1.0, 1.0, 0.0]
    - Rgba is Copy, Clone, Debug, PartialEq, Eq, Hash
    - Default::default() for Rgba is transparent black (0, 0, 0, 0)
  </behavior>
  <action>
Create src/color.rs implementing the Rgba type following the research document's code example:

1. Define `pub struct Rgba { pub r: u8, pub g: u8, pub b: u8, pub a: u8 }` with derives: Clone, Copy, Debug, Default, PartialEq, Eq, Hash

2. Implement constructors:
   - `pub const fn rgb(r: u8, g: u8, b: u8) -> Self` (alpha defaults to 255)
   - `pub const fn rgba(r: u8, g: u8, b: u8, a: u8) -> Self`
   - `pub fn from_f32(r: f32, g: f32, b: f32, a: f32) -> Self` (clamps 0.0..=1.0, multiplies by 255)

3. Implement convenience methods:
   - `pub fn to_f32_array(&self) -> [f32; 4]` (each component as f32 / 255.0)
   - `pub fn to_f32_tuple(&self) -> (f32, f32, f32, f32)`

4. Implement `std::fmt::Display`: output `#rrggbb` when a==255, `#rrggbbaa` otherwise. Always lowercase hex.

5. Implement `std::str::FromStr` with `type Err = String`:
   - Strip leading '#' if present
   - Match on hex length: 3 (RGB shorthand), 4 (RGBA shorthand), 6 (RRGGBB), 8 (RRGGBBAA)
   - For 3/4 char: double each digit (e.g., 'a' -> 0xaa, which is `val * 17`)
   - For 6/8 char: parse 2-char slices with `u8::from_str_radix`
   - Other lengths: return descriptive error
   - Invalid hex chars: return error from from_str_radix

6. Implement custom `Serialize` (serialize as hex string via Display) and `Deserialize` (deserialize from string via FromStr).

7. Write comprehensive tests in a `#[cfg(test)] mod tests` block covering all behaviors listed above including edge cases (empty input, invalid chars, wrong lengths, case insensitivity, alpha omission).

Write tests FIRST (RED), then implement to make them pass (GREEN).
  </action>
