<objective>
Normalize GTK-convention symbolic SVGs to use `currentColor` so that dark-on-dark icons on GNOME/GTK desktops are correctly recolored by existing connector colorize logic.

Purpose: GTK symbolic icons (Adwaita, Yaru, elementary) use hardcoded dark fill colors that GTK replaces at paint time. When native-theme loads these SVGs from disk, it returns raw bytes with no recoloring, making icons invisible on dark themes. This fix normalizes GTK foreground placeholders to `currentColor` at the source, leveraging existing connector infrastructure.

Output: Modified `freedesktop.rs` with `find_icon` returning `(PathBuf, bool)`, new `normalize_gtk_symbolic` function, updated load functions, and 9 unit tests.
</objective>
