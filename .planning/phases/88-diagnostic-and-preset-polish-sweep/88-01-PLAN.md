1. Define `DiagnosticEntry` enum (per POLISH-01, doc 1 section 23 Option B with per-entry typing):
```rust
/// A single diagnostic observation about platform theme support.
///
/// Returned by [`diagnose_platform_support()`]. Each variant represents
/// a specific category of diagnostic information. Use `Display` to get
/// a human-readable string, or pattern-match for programmatic inspection.
#[derive(Clone, Debug, PartialEq, Eq)]
#[non_exhaustive]
pub enum DiagnosticEntry {
    /// Platform identification (e.g. "Linux", "macOS", "Windows").
    Platform(&'static str),
    /// Detected desktop environment (Linux only).
    DesktopEnv(crate::detect::LinuxDesktop),
    /// An environment variable was read successfully.
    EnvVar { name: &'static str, value: String },
    /// An environment variable was missing or empty.
    EnvVarMissing(&'static str),
    /// An external tool was found and operational.
    ToolAvailable { name: &'static str, version: String },
    /// An external tool was found but returned an error.
    ToolError(&'static str),
    /// An external tool was not found on PATH.
    ToolMissing { name: &'static str, impact: &'static str },
    /// A config file was found at the given path.
    ConfigFound { name: &'static str, path: std::path::PathBuf },
    /// A config file was not found at the expected path.
    ConfigMissing { name: &'static str, path: std::path::PathBuf },
    /// A cargo feature is enabled.
    FeatureEnabled(&'static str),
    /// A cargo feature is disabled.
    FeatureDisabled { feature: &'static str, impact: &'static str },
}
```

Add `impl fmt::Display for DiagnosticEntry` that reproduces the current string output format
for each variant. For example:
- `Platform(p)` -> `"Platform: {p}"`
- `DesktopEnv(de)` -> `"Detected DE: {de:?}"`
- `EnvVar { name, value }` -> `"{name}: {value}"`
- `EnvVarMissing(name)` -> `"{name}: not set"`
- `ToolAvailable { name, version }` -> `"{name}: available ({version})"`
- `ToolError(name)` -> `"{name}: found but returned error"`
- `ToolMissing { name, impact }` -> `"{name}: not found ({impact})"`
- `ConfigFound { name, path }` -> `"{name}: found at {path}"`
- `ConfigMissing { name, path }` -> `"{name}: not found at {path}"`
- `FeatureEnabled(f)` -> `"{f} support: enabled"`
- `FeatureDisabled { feature, impact }` -> `"{feature} support: disabled ({impact})"`

2. Rewrite `diagnose_platform_support() -> Vec<DiagnosticEntry>` to push typed variants
instead of formatted strings. The logic stays the same, only the push calls change. For example:
- `diagnostics.push("Platform: Linux".to_string())` becomes `diagnostics.push(DiagnosticEntry::Platform("Linux"))`
- The XDG_CURRENT_DESKTOP block pushes `EnvVar` + `DesktopEnv` on success, `EnvVarMissing` + `DesktopEnv(LinuxDesktop::Unknown)` on failure
- gsettings check pushes `ToolAvailable`, `ToolError`, or `ToolMissing`
- KDE kdeglobals check pushes `ConfigFound` or `ConfigMissing`
- Feature checks push `FeatureEnabled` or `FeatureDisabled`

3. Define `PlatformPreset` struct (per POLISH-02, doc 1 section 24 Option C):
```rust
/// Structured information about the platform's default preset.
///
/// Returned by [`platform_preset_name()`]. The `name` field is the
/// user-facing preset name (e.g. `"macos-sonoma"`). The `is_live` field
/// indicates whether the preset is a live (geometry-only) preset used
/// by the OS-first pipeline.
///
/// `Display` returns the user-facing name.
#[derive(Clone, Debug, PartialEq, Eq)]
pub struct PlatformPreset {
    /// User-facing preset name (e.g. "kde-breeze", "adwaita", "macos-sonoma").
    pub name: &'static str,
    /// Whether this is a live preset (geometry-only merge base for OS readers).
    pub is_live: bool,
}
```

Add `impl fmt::Display for PlatformPreset` that returns `self.name`.

Add a method `pub fn live_name(&self) -> String` that returns `format!("{}-live", self.name)`
when `is_live` is true, or `self.name.to_string()` when not.

- Windows returns `PlatformPreset { name: "windows-11", is_live: true }`
- Linux KDE returns `PlatformPreset { name: "kde-breeze", is_live: true }`
- Linux other returns `PlatformPreset { name: "adwaita", is_live: true }`
- Unknown platform returns `PlatformPreset { name: "adwaita", is_live: true }`
