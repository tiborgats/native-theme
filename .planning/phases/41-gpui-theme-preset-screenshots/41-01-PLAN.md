<task type="auto">
  <name>Task 1: Create gpui screenshot capture script and update master orchestration</name>
  <files>scripts/generate_gpui_screenshots.sh, scripts/generate_assets.sh</files>
  <action>
Create `scripts/generate_gpui_screenshots.sh` that captures 4 gpui showcase screenshots using spectacle on KDE Wayland. The script must:

1. Set `SCRIPT_DIR`, `PROJECT_ROOT`, `OUTPUT_DIR="$PROJECT_ROOT/docs/assets"`, `DELAY=3`
2. Define the same 4 theme presets as iced: `THEMES=("dracula:dark" "nord:light" "catppuccin-mocha:dark" "macos-sonoma:light")`
3. Pre-build the showcase: `cargo build -p native-theme-gpui --example showcase --release`
4. Kill any stale spectacle: `pkill spectacle 2>/dev/null || true`
5. Add a trap to clean up on exit: `trap 'kill "$PID" 2>/dev/null || true' EXIT`
6. Loop over THEMES, for each:
   - Split on colon: `theme="${entry%%:*}"` and `variant="${entry##*:}"`
   - Output filename: `gpui-linux-${theme}-${variant}.png`
   - Launch showcase in background: `cargo run -p native-theme-gpui --example showcase --release -- --theme "$theme" --variant "$variant" --tab buttons &`
   - Capture PID: `PID=$!`
   - Wait for render: `sleep "$DELAY"`
   - Capture: `spectacle -i -a -b -n -e -o "$output_file"`
   - Wait for spectacle file write: `sleep 1`
   - Kill showcase: `kill "$PID" 2>/dev/null || true; wait "$PID" 2>/dev/null || true`
7. Print summary with count of generated files
8. Make the script executable (`chmod +x`)

Print a warning at the top of the capture loop: "Do not interact with the desktop during capture."

Then update `scripts/generate_assets.sh` to add a Step 3 calling the new gpui script:
```
echo "--- Step 3: Generating gpui showcase screenshots ---"
bash "$SCRIPT_DIR/generate_gpui_screenshots.sh"
echo ""
```

Insert this between the existing Step 2 and the final "All visual assets generated" message.
