  <action>
Replace the entire error.rs with the Option F structure from doc 1 section 31.2:

1. **Delete ThemeResolutionError struct** and all its impls (Display, Error, field_category).

2. **Add RangeViolation struct** (public, derives Debug, Clone):
   ```rust
   pub struct RangeViolation {
       pub path: String,
       pub value: f64,
       pub min: Option<f64>,
       pub max: Option<f64>,
   }
   ```
   Add Display impl: `"{path} must be {min}..={max}, got {value}"` (handling None for open-ended ranges).

3. **Add ErrorKind enum** (public, derives Debug, Clone, Copy, PartialEq, Eq, non_exhaustive):
   ```rust
   pub enum ErrorKind {
       Platform,
       Parse,
       Resolution,
       Io,
   }
   ```

4. **Rewrite Error enum** (9 variants, non_exhaustive, derives Debug only):
   ```rust
   pub enum Error {
       FeatureDisabled { name: &'static str, needed_for: &'static str },
       PlatformUnsupported { platform: &'static str },
       UnknownPreset { name: String, known: &'static [&'static str] },
       WatchUnavailable { reason: &'static str },
       Toml(toml::de::Error),
       Io(std::io::Error),
       ResolutionIncomplete { missing: Vec<String> },
       ResolutionInvalid { errors: Vec<RangeViolation> },
       ReaderFailed { reader: &'static str, source: Box<dyn std::error::Error + Send + Sync> },
   }
   ```

   Design notes:
   - `PlatformUnsupported { platform: &'static str }` uses &'static str (not an enum) because there is no Platform enum in the crate yet. The roadmap does not require one for Phase 71.
   - `Io(std::io::Error)` drops the Arc wrapping since Clone was removed in Phase 70. std::io::Error is already Send + Sync.
   - `ResolutionIncomplete { missing: Vec<String> }` keeps the stringly-typed paths (same as current ThemeResolutionError::missing_fields). The doc 1 mention of `Vec<FieldPath>` is a future refinement; Phase 71 keeps Vec<String> for compatibility.
   - `ReaderFailed` uses `Box<dyn std::error::Error + Send + Sync>` (not Arc) since Clone is gone.

5. **Add Error::kind() method** with exhaustive match (per doc 1 section 31.2 Option F lines 3631-3641). This is the compile-time enforcement: any new variant without a kind() branch is a compile error.

6. **Display impl**: Move ThemeResolutionError's field_category + categorized Display logic into the ResolutionIncomplete arm. The hint about from_toml_with_base stays. ResolutionInvalid gets its own display listing each RangeViolation.

7. **std::error::Error impl**: source() returns Some for Toml, Io, ReaderFailed. Returns None for all others (ResolutionIncomplete and ResolutionInvalid are data-carrying, not wrapping an inner error).

8. **From impls**:
   - `From<toml::de::Error>` -> Error::Toml (direct wrap, preserving spans)
   - `From<std::io::Error>` -> Error::Io (direct, no Arc)
   - Delete `From<toml::ser::Error>` -- the crate uses `to_string()` serialization paths that already handle errors differently. If compilation reveals it's still needed, add `From<toml::ser::Error>` mapping to a Toml variant or keep as string.

9. **Tests**: Write all tests from the behavior list above. Delete all existing ThemeResolutionError tests and old Error variant tests. Keep the send_sync test.
