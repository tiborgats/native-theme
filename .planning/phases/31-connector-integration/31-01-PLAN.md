<objective>
Add AnimatedIcon playback helpers to both toolkit connectors.

Purpose: Applications using gpui or iced need to convert AnimatedIcon data model values (from loading_indicator()) into toolkit-native animation primitives. The connectors bridge this gap with pure data-conversion helpers that follow the same pattern as existing to_image_source() and to_svg_handle() functions. Connectors do NOT own animation state -- timers, frame indices, and subscriptions remain application-level.

Output: Four new public functions across two connector crates (2 per connector), with unit tests.
</objective>
