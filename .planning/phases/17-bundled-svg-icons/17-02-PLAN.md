**2. Create `native-theme/src/model/bundled.rs`** with:

```rust
use super::icons::{IconRole, IconSet};

/// Returns the raw SVG bytes for a bundled icon, if the corresponding
/// feature flag is enabled.
///
/// Returns `None` when:
/// - The requested `IconSet` is not `Material` or `Lucide`
/// - The feature flag for the requested set is not enabled
///
/// Returns `Some(&[u8])` containing valid SVG bytes when the feature
/// is enabled. Callers typically convert to `IconData::Svg(bytes.to_vec())`.
#[allow(unreachable_patterns)]
pub fn bundled_icon_svg(set: IconSet, role: IconRole) -> Option<&'static [u8]> {
    match set {
        #[cfg(feature = "material-icons")]
        IconSet::Material => material_svg(role),

        #[cfg(feature = "lucide-icons")]
        IconSet::Lucide => lucide_svg(role),

        _ => None,
    }
}
```

Then implement `material_svg(role)` and `lucide_svg(role)` as private functions with `#[cfg(feature = "...")]` guards.

For `material_svg`, map ALL 42 IconRole variants using `include_bytes!`:
- Use path `"../../icons/material/{filename}.svg"` relative to bundled.rs
- DialogWarning -> warning.svg, DialogError -> error.svg, etc.
- TrashEmpty AND TrashFull both map to delete.svg
- StatusError reuses error.svg (same as DialogError)
- Help reuses help.svg (same as DialogQuestion)
- Use `#[allow(unreachable_patterns)]` for the `_ => return None` wildcard arm

For `lucide_svg`, map ALL 42 IconRole variants similarly:
- Use path `"../../icons/lucide/{filename}.svg"`
- TrashEmpty AND TrashFull both map to trash-2.svg
- StatusError reuses circle-x.svg (same as DialogError)
- Help reuses circle-question-mark.svg (same as DialogQuestion)

CRITICAL: Every single IconRole variant MUST have an explicit match arm. The success criteria requires "every IconRole variant resolves to valid SVG bytes." Do NOT use the wildcard `_` arm for any known variant -- only for #[non_exhaustive] forward compat.
