<objective>
Atomically rename the four core types per NAME-01: `ThemeSpec->Theme`, `ThemeVariant->ThemeMode`, `ResolvedThemeVariant->ResolvedTheme`, `ResolvedThemeDefaults->ResolvedDefaults`. `SystemTheme` remains unchanged.

Purpose: The v0.5.7 API vocabulary is confusing -- six names that all mean "theme" in different ways. The rename clarifies the hierarchy: `Theme` is the top-level spec, `ThemeMode` is a single light/dark mode, `ResolvedTheme` is the fully-resolved output, `ResolvedDefaults` is the defaults portion.

Output: All source files, tests, connectors, and README updated with new type names. Zero compilation errors, all tests pass.
</objective>
