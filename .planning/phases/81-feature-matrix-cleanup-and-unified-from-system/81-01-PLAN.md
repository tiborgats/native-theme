  **lib.rs changes:**

  1. Replace the `from_system()` method. The current implementation calls `pipeline::from_system_inner()`. Change it to use cfg-gated dispatch:

  On Linux -- use pollster to drive the async inner:
  ```rust
  #[cfg(target_os = "linux")]
  pub fn from_system() -> crate::Result<Self> {
      pollster::block_on(pipeline::from_system_inner())
  }
  ```

  On non-Linux (macOS/Windows) -- the async fn has zero .await points, so use a noop-waker single-poll to avoid pulling pollster as a dependency on non-Linux platforms:
  ```rust
  #[cfg(not(target_os = "linux"))]
  pub fn from_system() -> crate::Result<Self> {
      let waker = std::task::Waker::noop();
      let mut cx = std::task::Context::from_waker(&waker);
      let mut fut = std::pin::pin!(pipeline::from_system_inner());
      match fut.as_mut().poll(&mut cx) {
          std::task::Poll::Ready(result) => result,
          std::task::Poll::Pending => Err(crate::Error::PlatformUnsupported {
              platform: "unexpected async suspension",
          }),
      }
  }
  ```

  This is zero-dependency, correct (the future resolves immediately since there are no .await points on non-Linux), and satisfies the single-implementation requirement. `Waker::noop()` is stable since Rust 1.85, well within the workspace's `rust-version = "1.94.0"`.
