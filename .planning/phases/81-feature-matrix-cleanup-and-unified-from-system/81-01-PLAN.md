  Simplest approach that satisfies the "one implementation" requirement:

  In pipeline.rs, have the single `from_system_async_inner()` for ALL platforms. In lib.rs:

  ```rust
  #[cfg(target_os = "linux")]
  pub fn from_system() -> crate::Result<Self> {
      pollster::block_on(pipeline::from_system_async_inner())
  }

  #[cfg(not(target_os = "linux"))]
  pub fn from_system() -> crate::Result<Self> {
      // On non-Linux, the async inner has no await points.
      // Drive it with a minimal inline executor to avoid adding
      // pollster as a dependency on macOS/Windows.
      let mut fut = std::pin::pin!(pipeline::from_system_async_inner());
      match fut.as_mut().poll(&mut std::task::Context::from_waker(
          std::task::Wake::noop_wake_ref(),
      )) {
          std::task::Poll::Ready(result) => result,
          std::task::Poll::Pending => {
              // The non-Linux path never awaits, so Pending is unreachable.
              // If this somehow triggers, surface it as an error rather than panicking.
              Err(crate::Error::PlatformUnsupported {
                  platform: "unexpected async suspension",
              })
          }
      }
  }
  ```

  Wait, `std::task::Wake::noop_wake_ref()` doesn't exist in stable Rust. The noop waker was stabilized as `std::task::Waker::noop()` in Rust 1.85. Since the workspace has `rust-version = "1.94.0"`, this is available.

  ```rust
  #[cfg(not(target_os = "linux"))]
  pub fn from_system() -> crate::Result<Self> {
      let waker = std::task::Waker::noop();
      let mut cx = std::task::Context::from_waker(&waker);
      let mut fut = std::pin::pin!(pipeline::from_system_async_inner());
      match fut.as_mut().poll(&mut cx) {
          std::task::Poll::Ready(result) => result,
          std::task::Poll::Pending => Err(crate::Error::PlatformUnsupported {
              platform: "unexpected async suspension",
          }),
      }
  }
  ```

  This is zero-dependency, correct (the future resolves immediately since there are no .await points), and satisfies the single-implementation requirement.
