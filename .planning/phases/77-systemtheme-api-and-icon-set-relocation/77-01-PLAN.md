**1. Create ColorMode enum in model/mod.rs** (near the top, before ThemeMode):

```rust
/// Light or dark color mode preference.
///
/// Used by [`SystemTheme`](crate::SystemTheme) to indicate the OS color
/// mode and by [`SystemTheme::pick()`](crate::SystemTheme::pick) to select
/// a resolved variant.
///
/// # Examples
///
/// ```
/// use native_theme::theme::ColorMode;
///
/// let mode = ColorMode::Dark;
/// assert!(mode.is_dark());
/// ```
#[derive(Clone, Copy, Debug, PartialEq, Eq, Hash)]
#[non_exhaustive]
pub enum ColorMode {
    /// Light appearance.
    Light,
    /// Dark appearance.
    Dark,
}

impl ColorMode {
    /// Returns `true` if this is dark mode.
    #[must_use]
    pub fn is_dark(self) -> bool {
        matches!(self, Self::Dark)
    }
}
```

Add `ColorMode` to the existing `pub(crate) use model::{...}` re-export list in lib.rs so internal code can use `crate::ColorMode`.

Also add it to the `pub mod theme { pub use crate::model::*; }` re-export path (already covered since theme re-exports all of model).

**2. Migrate SystemTheme in lib.rs:**

- Replace `pub is_dark: bool` with `pub mode: ColorMode`
- Delete the `active()` method entirely
- Change `pick(&self, is_dark: bool)` to:
  ```rust
  /// Pick a resolved variant by color mode.
  ///
  /// # Examples
  ///
  /// ```no_run
  /// use native_theme::theme::ColorMode;
  ///
  /// let sys = native_theme::SystemTheme::from_system().unwrap();
  /// let dark = sys.pick(ColorMode::Dark);
  /// let active = sys.pick(sys.mode);
  /// ```
  #[must_use]
  pub fn pick(&self, mode: ColorMode) -> &ResolvedTheme {
      match mode {
          ColorMode::Light => &self.light,
          ColorMode::Dark => &self.dark,
      }
  }
  ```
  Note: the `#[non_exhaustive]` on `ColorMode` does NOT require a wildcard arm here because this is same-crate usage (per Phase 75-01 decision).
