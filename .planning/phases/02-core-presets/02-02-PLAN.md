Create `tests/preset_loading.rs` as an integration test file. Use `native_theme::` imports (crate root re-exports). Structure tests as individual #[test] functions, NOT a single mega-test. Use descriptive test names.

Test structure:
1. `all_presets_parse_without_error` -- iterate list_presets(), call preset() on each, unwrap
2. `all_presets_have_both_variants` -- assert light.is_some() && dark.is_some() for each
3. `all_presets_have_core_colors` -- check accent, background, foreground are Some in both variants for each preset
4. `all_presets_have_status_colors` -- check danger, warning, success are Some in both variants
5. `all_presets_have_interactive_colors` -- check selection, link are Some in both variants
6. `all_presets_have_valid_fonts` -- check family is Some, size is Some and > 0
7. `all_presets_have_geometry` -- check radius is Some
8. `all_presets_have_spacing` -- check m is Some
9. `all_presets_round_trip_toml` -- to_toml then from_toml, verify core.accent matches
10. `list_presets_returns_three_entries` -- assert_eq!(list_presets().len(), 3)
11. `dark_backgrounds_are_darker` -- for each preset, verify dark bg RGB sum < light bg RGB sum (catches copy-paste errors)
