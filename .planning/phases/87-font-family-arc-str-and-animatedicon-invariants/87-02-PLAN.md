<objective>
Migrate `FontSpec::family` from `String` to `Arc<str>` and `ResolvedFontSpec::family`
from `String` to `Arc<str>` across the core crate and both connectors.

Purpose: Eliminate ~22 separate owned String clones of the same font family name per
resolved theme. With `Arc<str>`, cloning shares a single heap allocation via reference
counting, reducing allocation pressure in widget-heavy layouts.

Output: `Arc<str>` family fields, serde `rc` feature enabled, connector access updated.
</objective>
