<objective>
Migrate `FontSpec::family` from `String` to `Arc<str>` and `ResolvedFontSpec::family`
from `String` to `Arc<str>` in the core crate type definitions, serde config, and
resolution logic.

Purpose: Eliminate ~22 separate owned String clones of the same font family name per
resolved theme. With `Arc<str>`, cloning shares a single heap allocation via reference
counting, reducing allocation pressure in widget-heavy layouts.

Output: `Arc<str>` family fields in core types, serde `rc` feature enabled, resolution
and inheritance logic updated.
</objective>

   #[test]
   fn resolved_font_clone_shares_family_arc() {
       let font = ResolvedFontSpec {
           family: Arc::from("Inter"),
           size: 14.0,
           weight: 400,
           style: FontStyle::Normal,
           color: crate::Rgba::rgb(0, 0, 0),
       };
       let cloned = font.clone();
       // str data pointers are identical -- same Arc allocation, not a deep copy
       assert!(std::ptr::eq(
           font.family.as_ref() as *const str,
           cloned.family.as_ref() as *const str,
       ));
