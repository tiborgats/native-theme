2. Create `FrameList` newtype with CUSTOM Deserialize (not derive):
```rust
/// A non-empty list of icon frames for animation.
///
/// Construction via `FrameList::new()` enforces the non-empty invariant.
/// Derefs to `&[IconData]` for ergonomic slice access.
#[derive(Debug, Clone, PartialEq, Eq, Serialize)]
pub struct FrameList(Vec<IconData>);

impl FrameList {
    /// Create a new FrameList, returning Err if frames is empty.
    pub fn new(frames: Vec<IconData>) -> Result<Self, EmptyFrameListError> {
        if frames.is_empty() {
            return Err(EmptyFrameListError);
        }
        Ok(Self(frames))
    }

    /// Returns the first frame (infallible — list is guaranteed non-empty).
    #[must_use]
    pub fn first(&self) -> &IconData {
        // SAFETY invariant: FrameList is only constructible via new() which rejects empty.
        // This is NOT unsafe — we use debug_assert as a belt-and-suspenders check.
        debug_assert!(!self.0.is_empty(), "FrameList invariant violated: empty list");
        &self.0[0] // This index is safe because FrameList::new rejects empty vecs
    }

    /// Returns the number of frames.
    #[must_use]
    pub fn len(&self) -> usize {
        self.0.len()
    }
}

impl Deref for FrameList {
    type Target = [IconData];
    fn deref(&self) -> &[IconData] {
        &self.0
    }
}
```

CRITICAL: Do NOT derive Deserialize on FrameList. Instead, implement a custom Deserialize
that calls FrameList::new() and rejects empty arrays (T-87-01 mitigation):
```rust
impl<'de> serde::Deserialize<'de> for FrameList {
    fn deserialize<D>(deserializer: D) -> Result<Self, D::Error>
    where
        D: serde::Deserializer<'de>,
    {
        let frames = Vec::<IconData>::deserialize(deserializer)?;
        FrameList::new(frames).map_err(|_| {
            serde::de::Error::custom("frame list must not be empty")
        })
    }
}
```
This ensures that `frames = []` in TOML/JSON produces a deserialization error, not a
FrameList that bypasses the non-empty invariant.

5. Change `AnimatedIcon` to use PRIVATE fields via wrapper structs.

Enum variant fields in Rust are always as-visible-as the enum itself -- there is no
way to make them private. To make fields genuinely inaccessible from outside the crate,
use wrapper structs with private fields as tuple variant payloads:

```rust
/// Data for a frame-based animation. Fields are private; use accessor methods.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub struct FramesData {
    frames: FrameList,
    frame_duration_ms: NonZeroU32,
}

/// Data for a transform-based animation. Fields are private; use accessor methods.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub struct TransformData {
    icon: IconData,
    animation: TransformAnimation,
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[non_exhaustive]
pub enum AnimatedIcon {
    /// A sequence of pre-rendered frames played at a fixed interval (loops infinitely).
    Frames(FramesData),
    /// A single icon with a continuous transform animation.
    Transform(TransformData),
}
```

With this design:
- `AnimatedIcon::Frames(FramesData { frames: ..., frame_duration_ms: ... })` fails to
  compile outside the crate because `FramesData` fields are private (pub struct, private fields).
- `AnimatedIcon::Frames(data)` works for pattern matching -- you get `data: &FramesData`
  and use accessor methods.
- Internal code (same crate) can still construct FramesData directly.

Add accessor methods on FramesData and TransformData:
```rust
impl FramesData {
    /// The animation frames (guaranteed non-empty).
    #[must_use]
    pub fn frames(&self) -> &FrameList { &self.frames }

    /// Duration of each frame in milliseconds (guaranteed non-zero).
    #[must_use]
    pub fn frame_duration_ms(&self) -> NonZeroU32 { self.frame_duration_ms }
}

impl TransformData {
    /// The icon being animated.
    #[must_use]
    pub fn icon(&self) -> &IconData { &self.icon }

    /// The transform animation applied to the icon.
    #[must_use]
    pub fn animation(&self) -> &TransformAnimation { &self.animation }
}
```

    /// Returns `Err(EmptyFrameListError)` if `frames` is empty.
    /// `frame_duration_ms` uses `NonZeroU32` so zero duration is prevented at the type level.
    pub fn frames(frames: Vec<IconData>, frame_duration_ms: NonZeroU32) -> Result<Self, EmptyFrameListError> {
