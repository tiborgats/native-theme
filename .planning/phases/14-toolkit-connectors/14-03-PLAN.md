    3. Create `connectors/native-theme-gpui/proposals/README.md`:
       This is the upstream PR proposal document (CONN-03). It should contain:

       **Title:** "Proposal: Per-Widget Metric Configuration in ThemeConfig"

       **Background:** gpui-component's ThemeConfig supports font_family, font_size, radius, radius_lg, shadow. However, per-widget sizing (button height, checkbox size, input padding, scrollbar width, slider track height, etc.) appears hardcoded in individual widget implementations.

       **Problem:** Theme systems like native-theme provide per-widget metrics from OS theme data (e.g., Windows GetSystemMetricsForDpi, KDE breezemetrics.h, macOS HIG). Without ThemeConfig hooks for these values, connector crates cannot apply native widget sizing.

       **Proposed Changes:** Add optional per-widget metric fields to ThemeConfig:
       - button_height, button_padding_horizontal
       - checkbox_size, checkbox_indicator_size
       - input_height, input_padding_horizontal
       - scrollbar_width
       - slider_track_height, slider_thumb_size
       - tab_height
       - tooltip_padding

       Each field should be `Option<f32>` so existing themes are unaffected (None = current hardcoded default).

       **How native-theme-gpui Would Use This:** Show mapping from WidgetMetrics sub-structs to proposed ThemeConfig fields.

       **Compatibility:** Fully backward-compatible since all new fields are optional.
