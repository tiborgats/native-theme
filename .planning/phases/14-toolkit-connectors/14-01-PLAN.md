<objective>
Build the iced connector core: map native-theme's ThemeColors to iced 0.14's Palette + Extended palette, and provide helper functions for widget metrics/geometry application.

Purpose: Developers using iced can convert any NativeTheme into an iced::Theme with a single function call, getting correct colors for all built-in widget styles. Widget metrics are exposed as helper functions for per-widget sizing.
Output: Working `native-theme-iced` crate with `to_theme()`, `pick_variant()`, palette mapping, extended palette overrides, and widget metric helpers.
</objective>

    1. Update `src/lib.rs` to declare modules and re-exports:
       ```rust

       pub mod extended;
       ```

    2. Implement `pub fn pick_variant<'a>(theme: &'a native_theme::NativeTheme, is_dark: bool) -> Option<&'a native_theme::ThemeVariant>`:
       - If is_dark: return theme.dark.as_ref().or(theme.light.as_ref())
       - If not is_dark: return theme.light.as_ref().or(theme.dark.as_ref())

    3. Implement `pub fn to_theme(variant: &native_theme::ThemeVariant, name: &str) -> iced::Theme`:
       - Call `palette::to_palette(variant)` to get base palette
       - Call `iced::Theme::custom_with_fn(name.to_string(), palette, |palette| { ... })`:
         - Inside closure: `let mut extended = iced::theme::palette::Extended::generate(palette);`
         - Call `extended::apply_overrides(&mut extended, variant);`
         - Return extended

    4. Implement widget metric helper functions (research says widget metrics like padding and min-height are set on widget instances, not through Catalog; provide helper fns):
       - `pub fn button_padding(variant: &native_theme::ThemeVariant) -> Option<[f32; 2]>` -- returns [horizontal, vertical] from widget_metrics.button
       - `pub fn input_padding(variant: &native_theme::ThemeVariant) -> Option<[f32; 2]>` -- from widget_metrics.input
       - `pub fn border_radius(variant: &native_theme::ThemeVariant) -> f32` -- from geometry.radius, default 4.0
       - `pub fn border_radius_lg(variant: &native_theme::ThemeVariant) -> f32` -- from geometry.radius_lg, default 8.0
       - `pub fn scrollbar_width(variant: &native_theme::ThemeVariant) -> f32` -- from geometry.scroll_width or widget_metrics.scrollbar.width, default 10.0
       These helpers let example apps and consumers apply native sizing without navigating nested Option types.

    5. Add tests for pick_variant (both light-preferred and dark-preferred, with fallback), to_theme (produces non-default theme), and helper functions.
