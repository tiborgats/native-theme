    1. Update `Cargo.toml` to add example with iced features needed for windowing:
       ```toml
       [[example]]
       name = "demo"
       required-features = []

       [dev-dependencies]
       iced = { version = "0.14", features = ["tokio"] }
       ```
       If the library already has iced as a dependency, the example can use it. But the example needs windowing features that the library dep may not include (since lib uses default-features = false). Add iced with windowing features under `[dev-dependencies]` so examples get the full feature set.

    2. Create `examples/demo.rs` as a standard iced application:

       **ThemeChoice enum:**
       - `OsTheme` and `Preset(String)` variants
       - Implement `Display` and `PartialEq`/`Eq`/`Clone`
       - `fn theme_choices() -> Vec<ThemeChoice>` lists OsTheme + all NativeTheme::list_presets()

       **App state:**
       - `current_choice: ThemeChoice`
       - `current_theme: iced::Theme`
       - `text_input_value: String`
       - `checkbox_checked: bool`
       - `slider_value: f32`
       - `progress_value: f32` (derived from slider)

       **Message enum:**
       - `ThemeSelected(ThemeChoice)`
       - `TextInputChanged(String)`
       - `CheckboxToggled(bool)`
       - `SliderChanged(f32)`
       - `ButtonPressed`

       **update():**
       - On ThemeSelected: load NativeTheme::preset() for Preset, or use NativeTheme::preset("default") for OsTheme (since from_system may not be available on all platforms). Pick the light variant (is_dark = false). Call native_theme_iced::to_theme() to create iced::Theme. Store both choice and theme.
       - On other messages: update corresponding state fields

       **view():**
       Layout in a centered, scrollable column with max-width ~600px:
       - Title text "native-theme-iced Demo"
       - Theme selector: `pick_list(theme_choices(), Some(current_choice), Message::ThemeSelected)`
       - Horizontal rule / separator
       - Section "Buttons": Row with primary button, secondary button (container styled), danger button
       - Section "Text Input": TextInput with placeholder
       - Section "Checkbox": Checkbox with label
       - Section "Slider": Slider 0.0..=100.0
       - Section "Progress Bar": ProgressBar showing slider value as progress
       - Section "Tooltip": Button wrapped in Tooltip
       - Section "Container": Container with border and padding showing sample text (demonstrates container styling)
       All wrapped in a Scrollable for the 8th widget type.

       Apply widget metric helpers where available:
       - Use `native_theme_iced::border_radius()` for container border radius if accessible
       - Use `native_theme_iced::button_padding()` for button padding if accessible

       **theme() method:**
       Return `self.current_theme.clone()`

    3. Ensure the example compiles with `cargo build -p native-theme-iced --example demo`.
       If iced feature flag issues arise (e.g., default-features = false in lib dep doesn't include required types), adjust the Cargo.toml dependency strategy. Options:
       - Use a feature flag on native-theme-iced: `features = ["iced-full"]` for examples
       - Or keep iced as full-featured in lib dep (simplest approach -- the connector is already tightly coupled to iced)
