  <files>native-theme/src/model/mod.rs, connectors/native-theme-gpui/src/lib.rs, connectors/native-theme-iced/src/lib.rs</files>
  <behavior>
    - pick_variant(true) with both variants returns dark
    - pick_variant(false) with both variants returns light
    - pick_variant(true) with only light variant returns light (fallback)
    - pick_variant(false) with only dark variant returns dark (fallback)
    - pick_variant with no variants returns None
  </behavior>
  <action>
**In `native-theme/src/model/mod.rs`:**

1. Add a method to the existing `impl NativeTheme` block:

```rust
/// Pick the appropriate variant for the given mode, with cross-fallback.
///
/// When `is_dark` is true, prefers `dark` and falls back to `light`.
/// When `is_dark` is false, prefers `light` and falls back to `dark`.
/// Returns `None` only if the theme has no variants at all.
pub fn pick_variant(&self, is_dark: bool) -> Option<&ThemeVariant> {
    if is_dark {
        self.dark.as_ref().or(self.light.as_ref())
    } else {
        self.light.as_ref().or(self.dark.as_ref())
    }
}
```

2. Add tests in the `#[cfg(test)] mod tests` block for the 5 behaviors listed above.

**In `connectors/native-theme-gpui/src/lib.rs`:**

1. Add `#[deprecated(since = "0.3.2", note = "Use NativeTheme::pick_variant() instead")]` to the existing `pick_variant` free function.
2. Replace its body to delegate: `theme.pick_variant(is_dark)`
3. Update the internal call in `to_theme` (or wherever `pick_variant` is called internally) to use `theme.pick_variant(is_dark)` directly instead of the free function, to avoid deprecation warnings in our own code.
4. Add `#[allow(deprecated)]` on the free function itself so the deprecated wrapper doesn't warn on its own body (it delegates to the method, so the deprecated annotation is only for external callers).

**In `connectors/native-theme-iced/src/lib.rs`:**

1. Same as gpui: add `#[deprecated(since = "0.3.2", note = "Use NativeTheme::pick_variant() instead")]` to `pick_variant`.
2. Replace body to delegate: `theme.pick_variant(is_dark)`
3. Update internal calls to use the method directly.
4. Add `#[allow(deprecated)]` on the free function.
