<objective>
Remove cross-set Material SVG fallback from all 4 locations (3 platform loaders + load_icon wildcard).

Purpose: Eliminate silent icon substitution. When a platform icon set does not have a mapping for a role, the loader should return None -- not silently substitute a Material SVG. This aligns the older loaders with the load_custom_icon design (Phase 22) which was built without cross-set fallback. Apps can make informed decisions about missing icons.

Output: 4 files updated with fallback removal, doc comments corrected, and tests rewritten.
</objective>

```rust
pub fn load_sf_icon(role: IconRole) -> Option<IconData> {
    let name = icon_name(IconSet::SfSymbols, role)?;
    let size = DEFAULT_ICON_SIZE;
    let image = load_symbol(name, size as f64)?;
    let cg_image = extract_cgimage(&image)?;
    let w = CGImage::width(&cg_image) as u32;
    let h = CGImage::height(&cg_image) as u32;
    let mut data = rasterize(&cg_image, w, h)?;
    unpremultiply_alpha(&mut data);
    Some(IconData::Rgba {
        width: w,
        height: h,
        data,
    })
}
```
