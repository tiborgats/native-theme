<objective>
Add the font_dpi field to the data model and implement the pt-to-px conversion resolution phase.

Purpose: This is the core fix for the font sizing bug. Without font_dpi, there is no mechanism to convert typographic points (from OS readers) to logical pixels (consumed by renderers). The conversion must happen in the resolution pipeline between Phase 1 (defaults internal) and Phase 2 (safety nets), before Phase 3 copies defaults to per-widget fonts.

Output: font_dpi field on ThemeDefaults and ResolvedThemeDefaults, resolve_font_dpi_conversion() method on ThemeVariant, updated resolve() pipeline.
</objective>

/// Convert a font size from typographic points to logical pixels.
fn convert_pt_to_px(size: &mut Option<f32>, dpi: f32) {
    if let Some(s) = size {
