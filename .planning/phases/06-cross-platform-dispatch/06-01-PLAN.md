<objective>
Implement cross-platform dispatch via from_system() that auto-detects the current platform and desktop environment, then calls the appropriate reader or returns a preset fallback.

Purpose: This is the primary public API for consumers who want "give me the current OS theme" without caring about platform details. It ties together all three readers (KDE, GNOME preset, Windows) behind a single sync entry point.

Output: from_system() function in lib.rs with Linux DE detection, platform dispatch, and comprehensive unit tests.
</objective>

Add the following to src/lib.rs, placed after the existing `pub use` re-exports and before the Result type alias:

1. Add `pub fn from_system() -> crate::Result<NativeTheme>` with unconditional `pub use` (no feature gate -- it is always available). Implementation uses nested `#[cfg(target_os)]` and `#[cfg(feature)]` blocks:
   - `#[cfg(target_os = "windows")]` block: inner `#[cfg(feature = "windows")]` returns `crate::windows::from_windows()`, inner `#[cfg(not(feature = "windows"))]` returns `Err(Error::Unsupported)`.
   - `#[cfg(target_os = "linux")]` block: returns `from_linux()`.
   - `#[cfg(not(any(target_os = "linux", target_os = "windows")))]` block: returns `Err(Error::Unsupported)`.

2. Add `#[cfg(target_os = "linux")] fn from_linux() -> crate::Result<NativeTheme>` (private). Calls `detect_linux_de()` on `std::env::var("XDG_CURRENT_DESKTOP").unwrap_or_default()`, then:
   - If `LinuxDesktop::Kde`: `#[cfg(feature = "kde")]` returns `crate::kde::from_kde()`. Without the feature, falls through to preset.
   - Fallback: `crate::preset("adwaita")` (this already returns `Result<NativeTheme>`, no conversion needed).
   - IMPORTANT: The `preset()` function returns `Result<NativeTheme>` (not `Option`). The research example using `.ok_or_else()` was incorrect -- just use `crate::preset("adwaita")` directly as the return value.

3. Add `#[cfg(target_os = "linux")] #[derive(Debug, PartialEq)] enum LinuxDesktop { Kde, Gnome, Unknown }` (crate-private, not pub).

4. Add `#[cfg(target_os = "linux")] fn detect_linux_de(xdg_current_desktop: &str) -> LinuxDesktop` (private pure function). Splits on ':' and checks each component for "KDE" then "GNOME", defaulting to Unknown.

5. Handle dead code warnings: When `kde` feature is not enabled, the `is_kde` check in `from_linux()` is unused. Structure the code so `detect_linux_de()` result is only bound when needed, or prefix with `_` via `let _de = ...` when no features use it. Cleanest approach: always call `detect_linux_de()` and match on it, with cfg-gated arms.

6. Add `#[cfg(test)] mod dispatch_tests` at the bottom of lib.rs with tests for:

   - `from_linux()` fallback: temporarily set XDG_CURRENT_DESKTOP to "GNOME", call from_linux(), verify it returns Ok with a NativeTheme that has name "adwaita". Use `unsafe { std::env::set_var(...) }` per Rust 2024 edition, and `#[serial]` is not needed since we test the pure function path primarily.
   - Actually, prefer testing from_linux() indirectly through the pure detect function + the preset path. The only integration-level test needed is verifying that from_linux() returns a valid NativeTheme for non-KDE desktops (which is just the Adwaita preset). Do NOT try to test from_kde() routing here since it would require a kdeglobals file.

IMPORTANT: Do NOT add `from_system` to any feature-gated `pub use` block. It must be unconditionally available. Add it as a standalone `pub use` or keep it as a bare `pub fn` directly in lib.rs (not in a submodule).

IMPORTANT: Use `#[cfg(target_os = "linux")]` attribute blocks (not `cfg!()` macro) so code that references kde/windows modules is removed at compile time on platforms where those modules do not exist.

IMPORTANT: Avoid `#[allow(unused)]` -- structure the code so warnings do not arise naturally.
