# Updating Platform Constants for New OS Versions

Brief intro explaining that native-theme ships platform-specific constants extracted from authoritative sources. When a new OS version ships (e.g., KDE Plasma 7, Windows 12, macOS 16, GNOME 48), these constants may need updating.

## KDE (Breeze)
- Source: `breezemetrics.h` from KDE Frameworks repository (https://invent.kde.org/plasma/breeze)
- File to update: `native-theme/src/kde.rs` (the `breeze_widget_metrics()` function)
- What to look for: changed default values for button height, checkbox size, scrollbar width, etc.
- Process: compare old and new `breezemetrics.h`, update corresponding constants, run `cargo test -p native-theme --features kde`

## Windows
- Source: Windows SDK headers, `GetSystemMetricsForDpi` API documentation, WinUI3 Fluent Design specs
- Files to update: `native-theme/src/windows.rs` (WinUI3 spacing constants, metric mappings)
- What to look for: new system metric indices, changed Fluent Design spacing values
- Process: check Windows SDK release notes, update constants, test on target Windows version

## macOS
- Source: Apple Human Interface Guidelines (HIG), AppKit release notes
- File to update: `native-theme/src/macos.rs` (the `macos_widget_metrics()` function)
- What to look for: changed default sizes in HIG (button heights, font sizes, control spacing)
- Process: review HIG for new macOS version, update constants, test on target macOS

## GNOME (Adwaita/libadwaita)
- Source: libadwaita source code (https://gitlab.gnome.org/GNOME/libadwaita), GTK4 source
- File to update: `native-theme/src/gnome.rs` (the `adwaita_widget_metrics()` function)
- What to look for: changed default values in libadwaita CSS/source
- Process: compare old and new libadwaita defaults, update constants, run `cargo test -p native-theme`

## Preset TOML Files
- After updating reader constants, also update the corresponding preset TOML files in `native-theme/src/presets/`
- Run the full test suite: `cargo test -p native-theme`

## Adding a New Platform
- Brief guidance on adding a new platform reader (new feature flag, module, reader function, preset files)

Keep the guide practical and concise (80-150 lines). Reference actual file paths in the codebase.
