<objective>
Phase 93-03 introduced a silent-ignore regression: `IconLoader::new(name).set(Freedesktop).theme("Adwaita").load()` silently drops the `.theme()` call because the `load()` method dispatches to `load_by_name()` which doesn't look at `self.freedesktop_theme`. The test `gnome_names_resolve_in_adwaita` has been failing since the Phase 93-03 migration but is masked by `run_check_soft` on the gpui crate.

Rather than patch the single dispatch path, this plan **eliminates the entire silent-ignore bug class** by restructuring `IconLoader` from one struct-with-per-set-optional-fields into five typed per-set loader structs. Each loader exposes ONLY the methods meaningful for its set. Calling `.theme()` on `SfSymbolsLoader` becomes a compile error (not a silent no-op). Adding a future set-specific option (SF Symbols weight, Material variant, Lucide stroke-width) cannot introduce a dispatch-bypass regression because the options live on exactly one type with no dispatch layer.

**Design rationale** (recorded here so future maintainers see the "why" in git history):

The old `IconLoader` struct had five fields, three of which were freedesktop-only: `size`, `fg_color`, `freedesktop_theme`. All three could be set on an `IconLoader` of any set; for non-freedesktop sets they were silently ignored. The `load()` dispatch on `id` variant (Role/Name/Custom) was an independent axis, and Phase 93-03 forgot to plumb `freedesktop_theme` through the `load_by_name()` branch, producing the observed regression. Five field × three variant × dispatch layer = 15 silent-drop possibilities; we had 15 latent traps.

The typed-per-set design reduces this to zero by making the set a type parameter (via separate structs per set). Set-specific options exist only on their owner type. The load-by-id-variant logic lives inside each type's `load()` method, which sees ALL relevant fields because they're fields of that type. No dispatch layer between "user sets option" and "implementation reads option."

Runtime set dispatch (needed in ~80% of connector call sites, which pick `IconSet` based on user choice) is handled by two free helper functions: `load_icon(id, set)` for default-options loads, and `load_icon_indicator(set)` for default-options spinner loads. Callers who need set-specific options write an explicit `match set { ... }` — which is verbose but forces the correct mental model (options ARE per-set).

**Why not a deprecation shim:** User explicitly said backwards-compatibility doesn't matter. v0.5.7 is pre-publication. Clean cut is honest and audit-able.

**Why not just patch `load_by_name`:** That fixes the one observed bug but leaves the silent-ignore design pattern in place. The next set-specific option to be added would face the same trap. The user asked for "most correct, very best solution" and explicitly accepted the larger migration cost.

**Why `load_freedesktop_spinner` also needs the theme argument:** A second latent bug surfaced during deeper inspection. Today's `load_freedesktop_spinner()` hardcodes `detect_theme()` — so even in the OLD API, calling `IconLoader::new(id).set(Freedesktop).theme("X").load_indicator()` silently ignored the theme override. Plumbing `theme: Option<&str>` through the spinner function closes that second hole.

// FreedesktopLoader — Linux / system-icons feature only
#[derive(Debug)]
#[must_use]
pub struct FreedesktopLoader<'a> {
    id: IconId<'a>,
    size: u16,
    fg_color: Option<[u8; 3]>,
    theme: Option<&'a str>,
}

impl<'a> FreedesktopLoader<'a> {
    /// Create a freedesktop loader. Defaults: size 24, no fg_color, system theme.
    pub fn new(id: impl Into<IconId<'a>>) -> Self {
        Self { id: id.into(), size: 24, fg_color: None, theme: None }
    }

    pub fn size(mut self, size: u16) -> Self { self.size = size; self }

    pub fn color(mut self, rgb: [u8; 3]) -> Self { self.fg_color = Some(rgb); self }

    pub fn color_opt(mut self, rgb: Option<[u8; 3]>) -> Self { self.fg_color = rgb; self }

    /// Override the freedesktop icon theme (e.g., "Adwaita", "breeze").
    /// When unset, uses the system-detected theme.
    pub fn theme(mut self, theme: &'a str) -> Self { self.theme = Some(theme); self }

    #[must_use]
    pub fn load(self) -> Option<IconData> {
        #[cfg(all(target_os = "linux", feature = "system-icons"))]
        {
            let theme_owned;
            let theme: &str = match self.theme {
                Some(t) => t,
                None => {
                    theme_owned = system_icon_theme();
                    &theme_owned
                }
            };
            match self.id {
                IconId::Role(role) => {
                    let name = icon_name(role, IconSet::Freedesktop)?;
                    crate::freedesktop::load_freedesktop_icon_by_name(name, theme, self.size, self.fg_color)
                }
                IconId::Name(name) => {
                    crate::freedesktop::load_freedesktop_icon_by_name(name, theme, self.size, self.fg_color)
                }
                IconId::Custom(provider) => {
                    if let Some(name) = provider.icon_name(IconSet::Freedesktop)
                        && let Some(data) = crate::freedesktop::load_freedesktop_icon_by_name(name, theme, self.size, self.fg_color)
                    {
                        return Some(data);
                    }
                    provider.icon_svg(IconSet::Freedesktop).map(IconData::Svg)
                }
            }
        }
        #[cfg(not(all(target_os = "linux", feature = "system-icons")))]
        {
            let _ = self;
            None
        }
    }

    /// Load the theme's animated process-working spinner.
    /// `theme` of `None` uses the system-detected theme; `Some(t)` overrides.
    #[must_use]
    pub fn load_indicator(theme: Option<&str>) -> Option<AnimatedIcon> {
        #[cfg(all(target_os = "linux", feature = "system-icons"))]
        { crate::freedesktop::load_freedesktop_spinner(theme) }
        #[cfg(not(all(target_os = "linux", feature = "system-icons")))]
        { let _ = theme; None }
    }
}

        IconSet::Material    => MaterialLoader::load_indicator(),
        IconSet::Lucide      => LucideLoader::load_indicator(),
