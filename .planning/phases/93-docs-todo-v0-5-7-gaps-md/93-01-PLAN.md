<objective>
G1 -- Remove `impl Default for Rgba` and the transitive `Default` derives on the three hand-written Resolved structs (`ResolvedBorderSpec`, `ResolvedFontSpec`, `ResolvedTextScaleEntry`). The manual `Rgba::default()` that returns `TRANSPARENT` is a footgun the §16 API review already flagged; Phase 90 kept it only because downstream Resolved structs required the bound via `require<T: Clone + Default>` in `validate_helpers.rs`. Rewriting those three fallback branches to use explicit sentinel values breaks the chain and lets us delete both the `Default` impl on `Rgba` and the `Default` derives on the three Resolved leaf structs.

Purpose: ship v0.5.7 without the "transparent black is the default colour" footgun the public API exposes, and without the bound-chain that forced us to keep it in Phase 90.

Output: four source files with reduced surface area, one rewritten validate_helpers module that constructs placeholders explicitly instead of via `T::default()`, and a green test+clippy+pre-release run.
