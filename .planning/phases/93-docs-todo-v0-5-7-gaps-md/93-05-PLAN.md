<objective>
G5 -- Drive non-widget `FIELD_NAMES` from a derive macro. Currently 7 hand-authored constants (font.rs: FontSpec/ResolvedFontSpec/TextScale[/TextScaleEntry]; border.rs: DefaultsBorderSpec/WidgetBorderSpec; defaults.rs: ThemeDefaults 35 entries; icon_sizes.rs: IconSizes) mirror the struct field names with drift caught only at test time. Direction B from the gap doc ("extend the inventory pattern we already have for widgets to non-widget structs") unifies them with the existing widget registry that `lint_toml` consumes.

Purpose: eliminate the ~174 field-name repetitions §I5 flagged. Renaming a struct field via `#[serde(rename = "foo_px")]` today requires a parallel FIELD_NAMES const update; after this plan, the derive handles it automatically.

Output: one new proc-macro in `native-theme-derive` (`ThemeFields` -- sister to `ThemeWidget`), 7 hand-authored `FIELD_NAMES` constants deleted, one `lint_toml` rewrite that consumes the unified inventory.

**Serde proxy complication:** `FontSpec` (font.rs:153-168) and `TextScaleEntry` (font.rs:253-264) declare `#[serde(try_from = "FontSpecRaw", into = "FontSpecRaw")]`. Their FIELD_NAMES mirror the RAW struct's serde fields, not the user struct's. Naive `#[derive(ThemeFields)]` on `FontSpec` would emit `["family", "size", "weight", "style", "color"]` which is WRONG — the correct list is `["family", "size_pt", "size_px", "weight", "style", "color"]`. The derive must either:
  (a) Detect the `try_from = "FontSpecRaw"` attribute and read fields from the named proxy type (hard — the proxy is in the same crate but the derive macro has no AST access at codegen time); OR
  (b) Accept a struct-level attribute `#[theme_layer(fields_from_proxy = "FontSpecRaw")]` that NAMES the proxy whose serde fields should be mirrored, and the macro reads those fields from a SECOND invocation of `parse_fields` on a user-provided sibling struct; OR
  (c) Accept a struct-level attribute `#[theme_layer(fields = "family, size_pt, size_px, weight, style, color")]` that explicitly lists the serialized names. Cleanest, least-magic. 
  
Pick (c) for clarity and correctness. The raw struct `FontSpecRaw` is a private serde-proxy implementation detail; its field list is a stable contract of the public FontSpec's serialization. Declaring it explicitly at the struct level makes the contract visible.
</objective>

       pub fn derive_theme_fields(input: TokenStream) -> TokenStream {
           let input = parse_macro_input!(input as DeriveInput);
           match derive_fields_inner(input) {
               Ok(tokens) => tokens.into(),
               Err(err) => err.to_compile_error().into(),
           }
       }

       fn derive_fields_inner(input: DeriveInput) -> syn::Result<proc_macro2::TokenStream> {
           let struct_name = &input.ident;
           let layer = parse::parse_layer_attrs(&input.attrs)?;
           let field_names: Vec<String> = if let Some(ref explicit) = layer.explicit_fields {
               explicit.clone()
           } else {
               // Introspect struct fields
               let data = match &input.data {
                   syn::Data::Struct(s) => s,
                   _ => return Err(syn::Error::new_spanned(&input.ident, "ThemeFields requires a struct")),
               };
               parse::parse_fields(&data.fields)?
                   .iter()
                   .map(|f| f.serde_rename.clone().unwrap_or_else(|| f.ident.to_string()))
                   .collect()
           };
           let struct_name_str = struct_name.to_string();
           let entries: Vec<proc_macro2::TokenStream> = field_names.iter().map(|n| quote::quote! { #n, }).collect();
           Ok(quote::quote! {
               const _: () = {
                   // Rather than emit a public `FIELD_NAMES` const (which collides with
                   // the existing widget-derive emission), submit directly to the registry.
                   inventory::submit! {
                       crate::resolve::FieldInfo {
                           struct_name: #struct_name_str,
                           field_names: &[#(#entries)*],
                       }
                   }
               };
           })
       }
