  Create src/windows.rs with:

  1. `fn win_color_to_rgba(c: ::windows::UI::Color) -> crate::Rgba` -- Converts windows Color {A, R, G, B} to Rgba::rgba(R, G, B, A).

  2. `fn is_dark_mode(fg: &crate::Rgba) -> bool` -- BT.601 luminance: 0.299*R + 0.587*G + 0.114*B. Returns true if luminance > 128.0 (light foreground = dark background = dark mode).

  3. `fn read_geometry() -> crate::ThemeGeometry` -- Calls unsafe GetSystemMetrics with SM_CXBORDER -> geometry.frame_width, SM_CXVSCROLL -> geometry.scroll_width. SAFETY comment: GetSystemMetrics is always safe to call with valid SM_* constants.

  4. `fn build_theme(accent: crate::Rgba, fg: crate::Rgba, bg: crate::Rgba, geometry: crate::ThemeGeometry) -> crate::NativeTheme` -- The testable core:
     - Calls is_dark_mode(&fg) to determine variant
     - Creates ThemeColors with: core.accent = Some(accent), core.foreground = Some(fg), core.background = Some(bg), interactive.selection = Some(accent), interactive.focus_ring = Some(accent), primary.background = Some(accent)
     - Creates ThemeVariant { colors, geometry, fonts: Default, spacing: Default }
     - Returns NativeTheme { name: "Windows", light/dark based on is_dark_mode }

  5. `pub fn from_windows() -> crate::Result<crate::NativeTheme>` -- Full implementation:
     - UISettings::new().map_err(|e| Error::Unavailable(format!("UISettings unavailable: {e}")))?
     - GetColorValue(UIColorType::Accent/Foreground/Background) -> win_color_to_rgba for each
     - read_geometry()
     - build_theme(accent, fg, bg, geometry)
     - Do NOT use UIColorType::Complement (throws runtime exception)

  6. `#[cfg(test)] mod tests` with 8 tests per behavior list above. Tests call build_theme and is_dark_mode directly with constructed Rgba values. No Windows APIs needed in tests.
