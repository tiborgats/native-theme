  <name>Windows Reader Testable Core</name>
  <files>src/windows.rs, Cargo.toml, src/lib.rs</files>
  <behavior>
    ## Feature flag setup
    - Cargo.toml: `windows = ["dep:windows"]` feature with `windows` crate dependency (version ">=0.58, <=0.62", optional, default-features = false, features = ["UI_ViewManagement", "Win32_UI_WindowsAndMessaging"])
    - src/lib.rs: `#[cfg(feature = "windows")] pub mod windows;` and `#[cfg(feature = "windows")] pub use windows::from_windows;`

    ## Helper: win_color_to_rgba
    - Converts a `::windows::UI::Color {A, R, G, B}` to `crate::Rgba::rgba(R, G, B, A)`
    - Note: this function uses the `::windows` crate type, so its test must be behind the feature gate

    ## Helper: is_dark_mode(fg: &Rgba) -> bool
    - Uses BT.601 luminance: 0.299*R + 0.587*G + 0.114*B
    - Returns true if luminance > 128.0 (light foreground = dark background = dark mode)
    - Test: white (255,255,255) foreground -> true (dark mode)
    - Test: black (0,0,0) foreground -> false (light mode)
    - Test: mid-gray (128,128,128) boundary -> luminance = 128.0, should return false (not > 128)

    ## Helper: read_geometry() -> ThemeGeometry
    - Calls unsafe GetSystemMetrics with SM_CXBORDER -> geometry.frame_width
    - Calls unsafe GetSystemMetrics with SM_CXVSCROLL -> geometry.scroll_width
    - SAFETY comment: GetSystemMetrics is always safe to call with valid SM_* constants

    ## Core: build_theme(accent, fg, bg, geometry) -> NativeTheme
    - Takes accent: Rgba, fg: Rgba, bg: Rgba, geometry: ThemeGeometry
    - Calls is_dark_mode(&fg) to determine variant
    - Creates ThemeColors with:
      - core.accent = Some(accent)
      - core.foreground = Some(fg)
      - core.background = Some(bg)
      - interactive.selection = Some(accent)
      - interactive.focus_ring = Some(accent)
      - primary.background = Some(accent)
    - Creates ThemeVariant { colors, geometry, fonts: Default, spacing: Default }
    - Returns NativeTheme { name: "Windows", light/dark based on is_dark_mode }
    - Test: dark mode (white fg) -> dark variant only, light is None
    - Test: light mode (black fg) -> light variant only, dark is None
    - Test: accent propagated to 4 fields (core.accent, interactive.selection, interactive.focus_ring, primary.background)
    - Test: geometry values preserved in output variant
    - Test: theme name is "Windows"

    ## Public API: from_windows() -> Result<NativeTheme>
    - Stub in Plan 01: calls UISettings::new() -> Error::Unavailable if fails
    - Plan 02 wires the full implementation
    - For Plan 01, include the full implementation since the structure is straightforward:
      - UISettings::new().map_err(|e| Error::Unavailable(format!("UISettings unavailable: {e}")))?
      - GetColorValue(Accent/Foreground/Background) -> win_color_to_rgba
      - read_geometry()
      - build_theme(accent, fg, bg, geometry)
    - Do NOT use UIColorType::Complement (throws runtime exception)

    ## Module naming
    - File is src/windows.rs, module is `mod windows`
    - Inside the module, use `::windows::` prefix for external crate references to avoid ambiguity
    - e.g., `use ::windows::UI::ViewManagement::{UISettings, UIColorType};`
