**Step 1: Create kdeglobals_path_pure function**

Add a new pure function alongside the existing kdeglobals_path():

```rust
/// Pure version of [`kdeglobals_path`] for testing.
///
/// Accepts XDG_CONFIG_HOME and HOME values as parameters instead of
/// reading environment variables, following the `_pure` suffix convention
/// (established in Phase 63).
#[cfg(test)]
fn kdeglobals_path_pure(xdg_config_home: Option<&str>, home: Option<&str>) -> std::path::PathBuf {
    if let Some(config_home) = xdg_config_home {
        if !config_home.is_empty() {
            return std::path::PathBuf::from(config_home).join("kdeglobals");
        }
    }
    if let Some(home) = home {
        return std::path::PathBuf::from(home)
            .join(".config")
            .join("kdeglobals");
    }
    // Last resort fallback
    std::path::PathBuf::from("/etc/xdg/kdeglobals")
}
```

Place this directly after the existing `kdeglobals_path()` function. Mark it `#[cfg(test)]` since it is only for tests.

IMPORTANT: Do NOT modify kdeglobals_path() itself. The production function keeps reading env vars. The pure variant is test-only.

Wait, this changes what is being tested. The original test verifies that from_kde() returns Error::ReaderFailed when the file is missing. That is testing from_kde()'s error wrapping. /// Read KDE theme from a specific kdeglobals file path (for testing).
```rust
#[cfg(test)]
pub(crate) fn from_kde_at(path: &std::path::Path) -> crate::Result<ThemeSpec> {
    let content = std::fs::read_to_string(path).map_err(|e| crate::Error::ReaderFailed {
        reader: "kde",
        source: Box::new(e),
    })?;
    from_kde_content(&content)
}
```
