1. **Header**: Crate name, one-line description ("Cross-platform native theme detection and loading for Rust GUI applications"), license badge (MIT OR Apache-2.0).

2. **Overview**: What native-theme is (toolkit-agnostic theme data model with 36 semantic color roles, TOML presets, optional OS theme reading). What it is NOT (not a GUI toolkit, not a widget library, adapters live in consumer code). Mention: works with egui, iced, slint, and any Rust GUI toolkit.

3. **Quick Start**: `cargo add native-theme` and a simple code example loading a preset and accessing fields. This example MUST be compile-tested (no `ignore` annotation):
```rust
let theme = native_theme::preset("dracula").unwrap();
let dark = theme.dark.as_ref().unwrap();
let accent = dark.colors.core.accent.unwrap();
let bg = dark.colors.core.background.unwrap();
// Convert to f32 for toolkits that use normalized colors
let [r, g, b, a] = accent.to_f32_array();
```

4. **Preset Workflow**: Show loading a base preset, creating sparse user overrides via from_toml(), and merging them. This example MUST be compile-tested:
```rust

let mut theme = preset("nord").unwrap();
let user_overrides = from_toml(r#"

[light.colors.core]
accent = "#ff6600"
"#).unwrap();
theme.merge(&user_overrides);
```

let theme = from_system().unwrap_or_else(|_| preset("default").unwrap());

6. **Toolkit Adapter Examples**: Three subsections (egui, iced, slint). Each shows a standalone `fn apply_theme(...)` mapping NativeTheme fields to the toolkit's API. Use `Visuals::dark()` as base.

   - **iced**: `rgba_to_iced` helper using `Color::from_rgb8(c.r, c.g, c.b)`, then `to_iced_theme` returning `Theme::custom("Native".into(), palette)` with Palette fields (background, text, primary, success, warning, danger).

   - **slint**: Show the .slint global singleton definition (`ThemeBridge` with color properties), then Rust code using `app.global::<ThemeBridge>()` with `slint::Color::from_argb_u8(c.a, c.r, c.g, c.b)` helper.

7. **Feature Flags**: Markdown table with columns: Feature, Enables, Platform, Notes. Cover all 5 features: `kde`, `portal`, `portal-tokio`, `portal-async-io`, `windows`. Explicitly note that `portal` alone is not useful -- must also enable `portal-tokio` or `portal-async-io`.

Group into Core (default, kde-breeze, adwaita), Platform (windows-11, macos-sonoma, material, ios), and Community (catppuccin-latte/frappe/macchiato/mocha, nord, dracula, gruvbox, solarized, tokyo-night, one-dark).

Mention that all fields are optional (Option<T>), unknown fields are ignored, and hex format is #RRGGBB or #RRGGBBAA.

10. **License**: "MIT OR Apache-2.0" with standard dual-license text.
