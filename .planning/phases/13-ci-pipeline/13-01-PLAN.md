<task type="auto">
  <name>Task 2: Create GitHub Actions CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/` directory and `.github/workflows/ci.yml` with the following structure:

**Trigger:** Push to `main` and `master` branches, and all pull requests.

**Env (workflow-level):** `CARGO_TERM_COLOR: always`

**Job 1: `fmt`** (fast gate, ~10s)
- Runner: `ubuntu-latest`
- Steps: checkout@v4, dtolnay/rust-toolchain@stable (components: rustfmt), `cargo fmt --all -- --check`
- No caching needed (no compilation)

**Job 2: `clippy`** (lint, depends on fmt)
- Runner: `ubuntu-latest`
- `needs: fmt`
- Env: `RUSTFLAGS: -Dwarnings`
- Steps: checkout@v4, dtolnay/rust-toolchain@stable (components: clippy), Swatinem/rust-cache@v2, `cargo clippy --workspace --all-targets`

**Job 3: `test`** (matrix, depends on fmt)
- `needs: fmt`
- `fail-fast: false`
- Matrix with `include` (7 entries, NOT a cross-product):
  - `{ name: "Linux (no features)", os: ubuntu-latest, flags: "--no-default-features" }`
  - `{ name: "Linux (kde)", os: ubuntu-latest, flags: "--features kde" }`
  - `{ name: "Linux (portal-tokio)", os: ubuntu-latest, flags: "--features portal-tokio" }`
  - `{ name: "Windows (no features)", os: windows-latest, flags: "--no-default-features" }`
  - `{ name: "Windows (windows)", os: windows-latest, flags: "--features windows" }`
  - `{ name: "macOS (no features)", os: macos-latest, flags: "--no-default-features" }`
  - `{ name: "macOS (macos)", os: macos-latest, flags: "--features macos" }`
- Job name: `Test ${{ matrix.name }}`
- Runner: `${{ matrix.os }}`
- Steps: checkout@v4, dtolnay/rust-toolchain@stable, Swatinem/rust-cache@v2, `cargo test -p native-theme ${{ matrix.flags }}`
- IMPORTANT: Use `-p native-theme` (not `--workspace --features`) because features are defined on the core crate only and `--workspace --features` fails when connector crates don't have those features.

**Job 4: `semver`** (independent, no dependencies)
- Runner: `ubuntu-latest`
- Steps: checkout@v4 with `fetch-depth: 0` (needs full git history for baseline-rev), obi1kenobi/cargo-semver-checks-action@v2 with `package: native-theme` and `baseline-rev: v0.1`
- Add a YAML comment: `# After publishing to crates.io, remove baseline-rev to auto-detect from registry`

Do NOT use any deprecated actions (actions-rs/*, hecrj/setup-rust-action). Do NOT use `--all-features` anywhere. Do NOT run fmt on multiple platforms.
  </action>
