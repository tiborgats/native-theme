<objective>
Define the SystemTheme type and rewire from_system()/from_system_async() to run the complete OS-first pipeline: reader -> preset merge -> resolve -> validate -> ResolvedTheme.

Purpose: Consumers calling from_system() currently get a raw NativeTheme with Option fields. After this plan, they get a SystemTheme with guaranteed-complete ResolvedTheme variants. This is the core API change of v0.5.0.

Output: SystemTheme struct with both resolved variants, platform preset mapping, rewired entry points, tests.

    ```rust
    /// Result of the OS-first pipeline. Holds both resolved variants.
    ///
    /// Produced by [`from_system()`] and [`from_system_async()`].
    /// Both light and dark are always populated: the OS-active variant
    /// comes from the reader + preset + resolve, the inactive variant
    /// comes from the preset + resolve.
    pub struct SystemTheme {
        /// Theme name (from reader or preset).
        pub name: String,
        /// Whether the OS is currently in dark mode.
        pub is_dark: bool,
        /// Resolved light variant (always populated).
        pub light: ResolvedTheme,
        /// Resolved dark variant (always populated).
        pub dark: ResolvedTheme,
        /// Pre-resolve light variant (retained for overlay support in PIPE-03).
        light_variant: ThemeVariant,
        /// Pre-resolve dark variant (retained for overlay support in PIPE-03).
        dark_variant: ThemeVariant,
    }
    ```

    Note: light_variant and dark_variant are private (pub(crate)) -- they exist for with_overlay() in Plan 02. Make them `pub(crate)` so Plan 02 can access them.

    Add methods:
    - `pub fn active(&self) -> &ResolvedTheme` -- returns dark if is_dark, else light

    Add internal helper `fn resolve_variant(mut variant: ThemeVariant) -> crate::Result<ResolvedTheme>`:
    ```rust
    fn resolve_variant(mut variant: ThemeVariant) -> crate::Result<ResolvedTheme> {
        variant.resolve();
        variant.validate()
    }
    ```

    Add internal `fn run_pipeline(reader_output: NativeTheme, preset_name: &str, is_dark: bool) -> crate::Result<SystemTheme>`:
    1. Load platform preset via `NativeTheme::preset(preset_name)?`
    2. Merge: `let mut merged = preset; merged.merge(&reader_output);`
    3. Keep reader name if non-empty, else use preset name
    4. Extract both light and dark variants: for whichever variant the reader provided, use the merged version; for the other, use the preset-only version. Both variants must exist after this step (default to ThemeVariant::default() if somehow both are missing from preset, but all presets have both).
    5. Store pre-resolve clones of both variants (for Plan 02 overlay support)
    6. Resolve and validate both variants
    7. Return SystemTheme

    GNOME note: GNOME's from_gnome() already merges the adwaita preset internally. The pipeline merges adwaita again via run_pipeline(), but this is harmless -- merge() is idempotent for matching values and reader values always win. No special case needed.

    Add a cross-platform helper to determine is_dark from reader output (used on macOS and Windows where `system_is_dark()` does not exist):

    ```rust
    /// Infer dark-mode preference from the reader's output.
    ///
    /// Returns `true` if the reader populated only the dark variant,
    /// `false` if it populated only light or both variants.
    /// On platforms that produce both variants (macOS), this defaults to
    /// `false` (light); callers can use [`SystemTheme::pick()`] for
    /// explicit variant selection regardless of this default.
    fn reader_is_dark(reader: &NativeTheme) -> bool {
        reader.dark.is_some() && reader.light.is_none()
    }
    ```

    **from_system():**
    ```rust
    pub fn from_system() -> crate::Result<SystemTheme> {
        #[cfg(target_os = "macos")]
        {
            #[cfg(feature = "macos")]
            {
                let reader = crate::macos::from_macos()?;
                let is_dark = reader_is_dark(&reader);
                return run_pipeline(reader, "macos-sonoma", is_dark);
            }
            #[cfg(not(feature = "macos"))]
            return Err(crate::Error::Unsupported);
        }
        #[cfg(target_os = "windows")]
        {
            #[cfg(feature = "windows")]
            {
                let reader = crate::windows::from_windows()?;
                let is_dark = reader_is_dark(&reader);
                return run_pipeline(reader, "windows-11", is_dark);
            }
            #[cfg(not(feature = "windows"))]
            return Err(crate::Error::Unsupported);
        }
        #[cfg(target_os = "linux")]
        { from_linux() }
        #[cfg(not(any(target_os = "linux", target_os = "windows", target_os = "macos")))]
        { Err(crate::Error::Unsupported) }
    }
    ```

    Note: macOS and Windows use `reader_is_dark()` to infer is_dark from the reader output. Linux continues to use the existing `system_is_dark()` which queries gsettings/kdeglobals. `reader_is_dark()` returns true only if the reader produced dark but not light, false otherwise. When macOS produces both variants, is_dark defaults to false -- users can always call `pick()` for explicit control.

    **from_linux():**
    Change return type to `Result<SystemTheme>`. Each branch now calls run_pipeline():
    ```rust
    fn from_linux() -> crate::Result<SystemTheme> {
        let is_dark = system_is_dark();
        let desktop = std::env::var("XDG_CURRENT_DESKTOP").unwrap_or_default();
        match detect_linux_de(&desktop) {
            #[cfg(feature = "kde")]
            LinuxDesktop::Kde => {
                let reader = crate::kde::from_kde()?;
                run_pipeline(reader, "kde-breeze", is_dark)
            }
            #[cfg(not(feature = "kde"))]
            LinuxDesktop::Kde => run_pipeline(NativeTheme::preset("adwaita")?, "adwaita", is_dark),
            LinuxDesktop::Gnome | LinuxDesktop::Budgie => {
                // GNOME sync path: no portal, just adwaita preset
                run_pipeline(NativeTheme::preset("adwaita")?, "adwaita", is_dark)
            }
            LinuxDesktop::Xfce | LinuxDesktop::Cinnamon | LinuxDesktop::Mate | LinuxDesktop::LxQt => {
                run_pipeline(NativeTheme::preset("adwaita")?, "adwaita", is_dark)
            }
            LinuxDesktop::Unknown => {
                #[cfg(feature = "kde")]
                {
                    let path = crate::kde::kdeglobals_path();
                    if path.exists() {
                        let reader = crate::kde::from_kde()?;
                        return run_pipeline(reader, "kde-breeze", is_dark);
                    }
                }
                run_pipeline(NativeTheme::preset("adwaita")?, "adwaita", is_dark)
            }
        }
    }
    ```
