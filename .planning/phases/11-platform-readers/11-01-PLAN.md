1. **Cargo.toml** -- Add the `macos` feature and its dependencies:
   ```toml
   [features]
   macos = ["dep:objc2", "dep:objc2-foundation", "dep:objc2-app-kit", "dep:block2"]

   [target.'cfg(target_os = "macos")'.dependencies]
   objc2 = { version = "0.6", optional = true }
   objc2-foundation = { version = "0.3", optional = true, features = ["NSString"] }
   objc2-app-kit = { version = "0.3", optional = true, features = [
       "NSColor", "NSColorSpace", "NSAppearance", "NSFont", "NSFontDescriptor",
       "objc2-core-foundation",
   ] }
   block2 = { version = ">=0.6.1, <0.8.0", optional = true }
   ```
   Use `[target.'cfg(target_os = "macos")'.dependencies]` so the ObjC crates are only resolved on macOS. Keep existing deps unchanged.

2. **native-theme/src/macos.rs** -- Create with the following structure:

   a. `nscolor_to_rgba(color: &NSColor, srgb: &NSColorSpace) -> Option<Rgba>` -- Convert NSColor to sRGB via `colorUsingColorSpace`, extract RGBA components as f32, return `Rgba::from_f32`. Handle `None` from `colorUsingColorSpace` gracefully.

   b. `read_semantic_colors() -> ThemeColors` -- Create `NSColorSpace::sRGBColorSpace()`, then map all ~20 NSColor semantic methods to ThemeColors fields using the mapping table from the interfaces section. Call each unsafe NSColor class method (e.g., `NSColor::controlAccentColor()`), pass through `nscolor_to_rgba`. Some fields share the same NSColor source (e.g., primary_background and accent both use controlAccentColor; danger_foreground, warning_foreground, etc. all use labelColor).

   c. `read_fonts() -> ThemeFonts` -- Read system font via `NSFont::systemFontOfSize(NSFont::systemFontSize())` and monospace via `NSFont::monospacedSystemFontOfSize_weight(size, NSFontWeight::Regular)`. Extract `familyName()` (returns Option<NSString>, convert to String) and `pointSize()` (CGFloat, cast to f32).

   d. `build_theme(light_colors: ThemeColors, dark_colors: ThemeColors, fonts: ThemeFonts) -> NativeTheme` -- This is the **testable core** (no OS API calls). Takes pre-read colors for both variants and fonts, constructs NativeTheme with name "macOS", light=Some(ThemeVariant{light_colors, fonts.clone(), Default geometry, Default spacing}), dark=Some(ThemeVariant{dark_colors, fonts, Default geometry, Default spacing}). Both variants always populated (unlike KDE/GNOME/Windows which populate only the active one).

   e. `pub fn from_macos() -> crate::Result<NativeTheme>` -- The public entry point. Uses `NSAppearance::appearanceNamed` with `"NSAppearanceNameAqua"` and `"NSAppearanceNameDarkAqua"`. For each appearance, calls `performAsCurrentDrawingAppearance` with a closure that calls `read_semantic_colors()`. Then calls `read_fonts()` once (fonts are appearance-independent in practice). Passes results to `build_theme()`. Returns `Error::Unavailable` if neither appearance can be created (unlikely but defensive).

   f. **Tests** (`#[cfg(test)] mod tests`) -- Test the `build_theme()` function (testable core) since actual NSColor/NSAppearance require macOS. Tests:
      - `build_theme_populates_both_variants`: Given distinct light/dark ThemeColors and fonts, verify both `light` and `dark` are `Some`, colors differ between variants, fonts are identical in both.
      - `build_theme_name_is_macos`: Verify `theme.name == "macOS"`.
      - `build_theme_fonts_populated`: Given ThemeFonts with family="SF Pro" and size=13.0, verify both variants have matching fonts.
      - `build_theme_geometry_and_spacing_default`: Verify both variants have default (empty) geometry and spacing.
      - `build_theme_colors_propagated_correctly`: Given light_colors with accent=Some(blue) and dark_colors with accent=Some(red), verify light.colors.accent==blue and dark.colors.accent==red.

      ```rust
      }
      ```

   e. Update from_system() doc comment to mention macOS behavior.
