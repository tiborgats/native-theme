<task type="auto">
  <name>Task 1: GNOME font reading and parse_gnome_font_string</name>
  <files>native-theme/src/gnome/mod.rs</files>
  <action>
Add GNOME font reading to gnome/mod.rs. This is self-contained and does not depend on other PLAT changes.

1. **`parse_gnome_font_string(s: &str) -> Option<(String, f32)>`** (new, pub(crate) for testing) -- Parse GNOME font format: gsettings outputs `'Family Name Size'` (with single quotes). Steps: trim single quotes from both ends, find last space via `rfind(' ')`, split into family (before space) and size (after space, parse as f32). Return None if: empty family, size <= 0.0, no space found, size not parseable.

2. **`read_gnome_fonts() -> ThemeFonts`** (new, pub(crate)) -- Read system UI font via `std::process::Command::new("gsettings").args(["get", "org.gnome.desktop.interface", "font-name"]).output()`. If success, parse stdout with `parse_gnome_font_string`. Then read monospace via `gsettings get org.gnome.desktop.interface monospace-font-name`. Return ThemeFonts with populated fields. Return `ThemeFonts::default()` if gsettings command not found or fails.

3. **Update `from_gnome()`** -- After building the theme from `build_theme()`, if the theme has a variant, merge the gnome fonts into it. Call `read_gnome_fonts()` and if it's not empty, update the variant's fonts:
   ```rust
   let fonts = read_gnome_fonts();
   if let Some(ref mut variant) = theme.light {
       variant.fonts.merge(&fonts);
   }
   if let Some(ref mut variant) = theme.dark {
       variant.fonts.merge(&fonts);
   }
   ```

4. **New tests** in the existing `#[cfg(test)] mod tests` block:
   - `parse_gnome_font_string_standard`: `"'Cantarell 11'"` -> Some(("Cantarell", 11.0))
   - `parse_gnome_font_string_multi_word`: `"'Noto Sans 10'"` -> Some(("Noto Sans", 10.0))
   - `parse_gnome_font_string_no_quotes`: `"Ubuntu Mono 13"` -> Some(("Ubuntu Mono", 13.0))
   - `parse_gnome_font_string_fractional_size`: `"'Inter 10.5'"` -> Some(("Inter", 10.5))
   - `parse_gnome_font_string_empty`: `""` -> None
   - `parse_gnome_font_string_only_quotes`: `"''"` -> None
   - `parse_gnome_font_string_no_size`: `"'Cantarell'"` -> None (no space = no split point)
   - `parse_gnome_font_string_zero_size`: `"'Font 0'"` -> None (size <= 0)
   - `parse_gnome_font_string_negative_size`: `"'Font -1'"` -> None (size <= 0 or parse fails)

<task type="auto">
  <name>Task 2: KDE+portal overlay, portal backend detection, from_linux() fallback</name>
  <files>native-theme/src/gnome/mod.rs, native-theme/src/kde/mod.rs, native-theme/src/lib.rs</files>
  <action>
1. **kde/mod.rs** -- Make `from_kde_content` accessible for reuse:
   - Change `fn from_kde_content` from private to `pub(crate) fn from_kde_content` so it can be called from lib.rs for the kdeglobals fallback path.
   - No other changes to kde/mod.rs.

2. **gnome/mod.rs** -- Add from_kde_with_portal() and detect_portal_backend():

   a. **`pub async fn from_kde_with_portal() -> crate::Result<NativeTheme>`** (new) -- Read KDE kdeglobals as base via `crate::kde::from_kde()`, then attempt async portal accent overlay. Steps:
      - Call `crate::kde::from_kde()?` for the base theme.
      - Try to connect to portal: `ashpd::desktop::settings::Settings::new().await`.
      - If portal available, try `settings.accent_color().await`.
      - If accent color available and valid (via `portal_color_to_rgba`), build an overlay NativeTheme with accent color applied to accent, selection, focus_ring, primary_background fields in the same variant(s) as the base theme.
      - Call `base.merge(&overlay)` to apply the accent overlay.
      - Return the merged result.
      - If portal unavailable or accent not available, return the KDE-only base (no error).
      This requires both `kde` and `portal` features. Gate with `#[cfg(feature = "kde")]`.

   b. **`pub(crate) async fn detect_portal_backend() -> Option<super::LinuxDesktop>`** (new) -- Detect which portal backend is running by checking D-Bus activatable names. Use `zbus::Connection::session().await` and `zbus::fdo::DBusProxy::new(&connection).await` then `list_activatable_names().await`. Search for names containing `portal.desktop.kde` -> return `Some(LinuxDesktop::Kde)`, `portal.desktop.gnome` -> return `Some(LinuxDesktop::Gnome)`. Return None if D-Bus not available or no matches.
      Note: `zbus` is a transitive dependency via ashpd -- do NOT add it as a direct dependency. Instead, use `ashpd`'s re-exported zbus if available, or add `zbus` to Cargo.toml only if needed for compilation. Check if ashpd re-exports it; if not, add `zbus = { version = "5", optional = true }` gated on the `portal` feature.

3. **lib.rs** -- Enhance from_linux() and detect_linux_de():

   a. **from_linux()** -- Add kdeglobals fallback path (PLAT-13). After the existing `match detect_linux_de(...)`:
      - For `LinuxDesktop::Unknown` (currently returns adwaita preset), check if kdeglobals file exists using `crate::kde::kdeglobals_path()` and `std::path::Path::exists()`. If it exists AND the `kde` feature is enabled, call `crate::kde::from_kde()`. If that fails or `kde` feature not enabled, fall back to adwaita preset.
      - The `LinuxDesktop::Gnome` arm stays as adwaita preset (users should call from_gnome() directly for live GNOME data since it's async).
      - The logic should be:
      ```rust
      LinuxDesktop::Gnome => NativeTheme::preset("adwaita"),
      LinuxDesktop::Unknown => {
          #[cfg(feature = "kde")]
          {
              let path = crate::kde::kdeglobals_path();
              if path.exists() {
                  return crate::kde::from_kde();
              }
          }
          NativeTheme::preset("adwaita")
      }
      ```

   b. **No changes to detect_linux_de()** for now -- D-Bus backend detection (PLAT-11) is async and the sync `from_linux()` cannot call it. The detect_portal_backend() function is exposed for async consumers (like from_gnome() or from_kde_with_portal()). The DE heuristic improvement is that async consumers can call detect_portal_backend() when XDG_CURRENT_DESKTOP returns Unknown.

   c. Add re-export: `#[cfg(all(feature = "portal", feature = "kde"))] pub use gnome::from_kde_with_portal;`
