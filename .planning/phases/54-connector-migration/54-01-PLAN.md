Replace 5 derive.rs color computations in gpui colors.rs with direct reads from the resolved theme's per-widget interactive state fields, and remove the dead `surface` field (K-4).

Purpose: The theme now provides explicit hover/active/selection colors on per-widget structs (added in Phases 52-53). Using these directly instead of computing at runtime means connector colors exactly match what the preset author intended, rather than approximating via blend/darken algorithms.

Output: Updated colors.rs with 5 direct theme reads, removed dead code, updated derive.rs doc comment, updated test assertions.
