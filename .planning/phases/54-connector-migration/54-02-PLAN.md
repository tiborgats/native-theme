1. **K-2: Change from_system() return type** from `Result<(Theme, ResolvedThemeVariant)>` to `Result<(Theme, ResolvedThemeVariant, bool)>`. Update the function signature (line 229) and the return statement (line 235):
   ```rust
   pub fn from_system() -> native_theme::Result<(Theme, ResolvedThemeVariant, bool)> {
       let sys = SystemTheme::from_system()?;
       let is_dark = sys.is_dark;
       let name = sys.name;  // K-5: move instead of clone
       let resolved = if is_dark { sys.dark } else { sys.light };
       let theme = to_theme(&resolved, &name, is_dark);
       Ok((theme, resolved, is_dark))
   }
   ```

2. **K-5: Change clone to move** -- `let name = sys.name;` instead of `let name = sys.name.clone();` (line 232). This works because `sys.name` is a `String` and `sys` fields are consumed individually (`sys.dark` or `sys.light` on the next line). Note: Rust allows partial moves from owned structs. The `sys.is_dark` (Copy) is read first, then `sys.name` (move), then `sys.dark`/`sys.light` (move). After this, `sys` is partially moved and cannot be used again, which is fine since we don't use it.

3. **Update module-level doc comment** (lines 1-69). Change the Quick Start example from:
   ```rust
   let (theme, resolved) = from_system()?;
   ```
   to:
   ```rust
   let (theme, resolved, is_dark) = from_system()?;
   ```
   Do the same for the `/// # Examples` doc comment on `from_system()` (around line 226).

4. **Update from_preset doc example** in the module-level doc comment (line 10-11) to also show the display-name fix:

   let (theme, resolved) = from_preset("catppuccin-mocha", true)?;
